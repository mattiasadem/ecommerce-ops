## Format specification — how the 25+25+20 templates are structured

Each template ships in **6 sub-fields** that the operator pastes into Amazon Seller Central / Walmart Seller Center / Target Plus Roundel / Amazon EU Seller Central / bol Seller Central:

1. **Title** (per-marketplace character-limit; pre-validated against Pitfall #2) — Amazon US: ≤200 chars / Walmart: ≤75 chars / Target Plus: ≤100 chars / Amazon EU: ≤50 chars mobile-first / bol: ≤150 chars. Example for Amazon US Default-voice: `Premium Stainless Steel Water Bottle — 32oz Insulated, BPA-Free, Keeps Drinks Cold 24hrs/Hot 12hrs — Sustainably Sourced, Leak-Proof Lid for Gym/Hike/Office (Black)`
2. **5-bullet-point set** (Amazon allows ≤500 chars/bullet; canonical high-CVR pattern is 200–300 chars per bullet) — each bullet has a 2-line format: ALL-CAPS BENEFIT KEY on line 1, 1-2 line narrative on line 2. Example for Amazon US Default-voice Bullet 1:
   - `PREMIUM STAINLESS STEEL CONSTRUCTION — Made from food-grade 18/8 stainless steel that's built to last a lifetime. No metallic aftertaste, no rust, no leaks — just clean hydration from the first sip to the last.`
3. **A+ content skeleton** (Brand-Registry-required for Amazon; Target Plus requires Roundel-content-block; ≤5 modules per Amazon's 2024 A+ content guide) — module-1 hero-image + headline, module-2 3-feature comparison chart, module-3 lifestyle-image grid, module-4 brand-story module, module-5 cross-sell module. Each module ships as a paste-ready HTML skeleton with the canonical image-dimension table + recommended brand-asset-upload paths.
4. **Search terms** (Amazon backend keywords ≤250 bytes; NOT visible on the PDP, but the canonical keyword-stuffing guardrail applies — must not include competitor-brand-names + must not duplicate title/bullet keywords) — Example for Amazon US Default-voice: `insulated water bottle, stainless steel bottle, leak proof water bottle, bpa free bottle, gym water bottle, hiking water bottle, office water bottle`
5. **Per-marketplace translation / regulatory-compliance flag** (for Amazon EU DE + FR + IT + ES + NL + bol NL) — a per-locale translation recipe + per-locale regulatory language (CE marking + GPSR effective Dec 2024 + EU compliance-pack language) + per-locale search-keyword research recommendation (do NOT literal-translate the US English keywords; use Helium 10 + Perpetual Traffic per-marketplace keyword-data).
6. **Voice-driven override syntax** (for multi-voice operators using one marketplace listing that branches on `voice_profile`) — Example Klaviyo-style conditional syntax: `{% if voice_profile == "Sustainable" %}...Sustainable variant title...{% elif voice_profile == "Gen-Z" %}...Gen-Z variant title...{% else %}...Default variant title...{% endif %}` (note: most marketplaces don't have native Klaviyo-style conditional blocks; multi-voice operators use the marketplace's variant-ASIN feature to ship different voice-column listings under different ASINs on the same parent listing).

**Amazon Seller Central variable substitution cheat sheet** (referenced across all 25+25+20 templates):

- `{{ product_name }}` — the canonical product name from Seller Central's Product Detail Page
- `{{ brand_name }}` — the brand name registered in Amazon Brand Registry
- `{{ color_variant }}` — the color-variant name (Black, Stainless, Matte Navy, etc.)
- `{{ size_variant }}` — the size-variant name (32oz, 24oz, 40oz)
- `{{ bullet_1_benefit }}` — the ALL-CAPS BENEFIT KEY for bullet 1
- `{{ bullet_1_narrative }}` — the 1-2 line narrative for bullet 1
- `{{ a_plus_hero_headline }}` — the A+ content module 1 hero-headline
- `{{ search_terms_csv }}` — the comma-separated backend-search-terms list
- `{{ locale }}` — Amazon EU locale (DE, FR, IT, ES, NL)
