## When to use this skill

Use this skill when the operator is doing **≥500 orders/month** AND/OR has a **chargeback-rate >0.65% (the canonical Visa VAMP threshold)** AND/OR is **losing $20k+/year to fraud + chargebacks + false-declines + card-network-fines + manual-chargeback-representation-costs** — the canonical "we're paying for chargebacks we should have prevented" anti-pattern. Specific signals: chargeback-rate has crossed 0.50% (the canonical pre-VAMP warning-zone per Visa 2024 + Mastercard 2024 benchmarks), false-decline-rate is 8–15% (the canonical "we're blocking 1 in 10 good orders" lost-revenue per Signifyd 2024 + Stripe Radar 2024 benchmarks), the operator is manually reviewing the order queue in Shopify and blocking 3–5% of orders on gut-feel (the canonical "we're rejecting good orders to avoid bad ones" anti-pattern), the brand sells $500+ AOV jewelry/watches/electronics and is losing 2–4% of orders to BIN-attack-fraud + synthetic-identity-fraud (the canonical "high-AOV-CNP-fraud-ring attacking the catalog" pattern per Signifyd 2024 + Veriff 2024 benchmarks), the brand sells consumables with subscription (Move #11) and is losing 0.5–2.0% of recurring-revenue to subscription-fraud (the canonical "friendly-fraud-on-recurring-charges" anti-pattern per Recharge 2024 + Stripe 2024 + Signifyd 2024 benchmarks), the brand is international (Move #22) and is losing 1.5–3.5% of cross-border-revenue to cross-border-CNP-fraud + package-redirection-fraud (the canonical "international-orders-2×-higher-fraud-rate" per Signifyd 2024 + Riskified 2024 + Visa 2024 benchmarks), the brand is on Shopify Payments and sees $0.40/order Shopify-Protect-as-upsell-or-the-order-is-going-to-cost-$50+ in chargeback-fees (the canonical "we didn't enable Shopify Protect" anti-pattern per Shopify 2024 benchmarks), or the operator cannot answer "what is our chargeback-rate today and how does it compare to the Visa-VAMP-threshold?" within 30 seconds (the canonical "we don't have a fraud-dashboard" anti-pattern). Move #23 is the **canonical risk-mitigation next-step** after Move #3 (checkout audit) + Move #6 (attribution) + Move #12 (3PL) + Move #21 (sales tax) — fraud is the structural-margin-leak that compounds as the brand scales; brands doing $5M GMV without automated fraud-screening lose $50k–$150k/year to fraud + chargebacks + false-declines + card-network-fines per Signifyd 2024 + NoFraud 2024 + Riskified 2024 + Visa 2024 + Mastercard 2024 benchmarks.

You have:

- **Shopify (or Ikas / BigCommerce / WooCommerce / headless) DTC store** with admin API access AND the Orders API + Payments API + Customers API enabled (Shopify Plus OR Shopify Advanced with the necessary scopes for chargeback-webhook + fraud-signal-export).
- **≥500 orders/month processed in the last 30 days** AND/OR **chargeback-rate >0.65% (Visa VAMP threshold)** AND/OR **AOV >$50** — below these thresholds, defer to Shopify native fraud-analysis + manual-review until volume justifies the platform fee. The canonical pre-shipping signal is "the cost of the platform fee exceeds the cost of the fraud" — at <500 orders/mo Path A (Stripe Radar + Shopify native fraud-analysis) is sufficient.
- **Move #3 (checkout audit) + Move #6 (Triple Whale attribution) + Move #12 (3PL) + Move #21 (sales tax) shipped and producing real data** — the fraud-screening engine reads attribution data to compute per-source-per-cohort fraud-rate (paid-social-cohort-A might have 1.2% fraud-rate vs organic-cohort-B at 0.3%) and reads fulfillment-data to detect address-mismatch-fraud + package-redirection-fraud. Without these upstream, the engine is screening against insufficient context.
- **A documented fraud-policy** — auto-approve threshold + auto-decline threshold + manual-review threshold + 3DS2-orchestration-rule (auto-trigger for orders >$200 OR new-customer OR international) + AVS/CVV-mismatch-handling + policy for high-risk-verticals (jewelry/watches/electronics/CBD) + policy for high-risk-cohorts (paid-social-first-time-customer-international). Without this, the engine has no foundation to score against. The canonical template is: `auto_approve = score < 25 AND amount < $200 AND customer_age > 7 days; auto_decline = score > 85 OR (amount > $1,000 AND customer_age < 1 day); manual_review = 25 ≤ score ≤ 85 OR (amount > $500 AND customer_age < 24 hours); 3DS2_trigger = score > 50 OR amount > $200 OR new_customer OR international`. The fraud-policy document ships in Pillar 1 Week 1.

## What "best in class" looks like

A best-in-class fraud + chargeback management engine has all 5 of these properties:

1. **5-pillar engine-selection framework.** Path A (Stripe Radar built-in + Shopify native fraud-analysis + Shopify Protect $0.40/order) for <$1M GMV Shopify-native brands on Stripe Payments; Path B (Signifyd $2,000/mo DEFAULT for $1M–$5M GMV Shopify-DTC apparel/beauty/home + NoFraud $999/mo alternative Path B); Path B+ (Kount custom + Sift custom + Forter custom) for $3M+ GMV brands needing persona-linking + device-fingerprinting + behavioral-analytics + account-takeover-protection at enterprise-scale; Path C (Riskified custom + Adyen RevenueProtect built-in for Adyen brands) for $5M+ GMV brands needing full-chargeback-liability-shift + policy-aware-decisioning + account-takeover-protection + multi-PSP-risk-orchestration; Path D (Veriff $1+/verification + Signifyd + Chargeflow) for $5M+ GMV high-AOV ($500+) brands needing biometric-ID-verification + selfie-match + liveness-detection + document-verification for jewelry/watches/electronics SKUs; Path E (Shopify Protect $0.40/order + Chargeflow $499/mo + MidMetrics $499/mo) for $1M–$5M GMV brands that want to stay on Shopify-native-stack with Chargeflow-dispute-recovery; Path F (custom-build + Stripe Radar + Klaviyo + Gorgias + Triple Whale) for $25M+ enterprise brands with multi-PSP-multi-region-multi-vertical orchestration. Engine selection drives everything downstream — wrong path picks cost 5–10× over a 12-month horizon.

2. **5-pillar framework applied in this order:**

   - **Pillar 1 — Fraud-policy document + auto-approve/auto-decline/manual-review threshold tuning (Week 1, 4 hr).** Document auto-approve threshold (score < 25) + auto-decline threshold (score > 85) + manual-review threshold (25 ≤ score ≤ 85) + 3DS2-orchestration-rule (auto-trigger for orders >$200 OR new-customer OR international) + AVS/CVV-mismatch-handling (allow if AVS-mismatch + CVV-match; decline if AVS-mismatch + CVV-mismatch + new-customer) + policy for high-risk-verticals (jewelry/watches/electronics/CBD decline if score > 60) + policy for high-risk-cohorts (paid-social-first-time-customer-international manual-review). Wire the policy to Shopify Flow + the fraud-screening-engine + the 3DS2-orchestration engine. Without this guardrail, the fraud-screening-engine will either over-decline (lost-revenue) or under-decline (chargeback-rate spikes).
   - **Pillar 2 — Fraud-screening-engine selection + per-vertical tuning + data-signal-orchestration (Week 1–2, 8 hr).** Signifyd / NoFraud / Riskified / Stripe Radar / Kount / Sift / Forter / Adyen RevenueProtect each bring 100–1,400 data-signals per order (device-fingerprint + IP-reputation + email-age + phone-reputation + BIN-risk + shipping-address-validation + billing-address-validation + behavioral-biometrics + persona-linking + dark-web-email-leak-check + chargeback-history-on-this-payment-fingerprint + proxy-detection + high-risk-country-detection + velocity-checks). Per-vertical tuning: apparel/beauty/home = lower-fraud-rate, baseline-policy-suffices; jewelry/watches/electronics/CBD = higher-fraud-rate, tighten-thresholds + add-Veriff-ID-verification-for-orders-$500+. The data-signal-orchestration reads from Shopify + Klaviyo + Triple Whale + the 3PL (Shipbob/ShipMonk/Stord address-validation) to compute the per-order risk-score.
   - **Pillar 3 — 3-D-Secure-2 (3DS2) + Visa Account-Name-Verification (AVN) + dynamic-step-up-authentication (Week 2–3, 6 hr).** 3DS2 is the canonical CNP-fraud-prevention protocol per Visa 2024 + Mastercard 2024 + EMVCo 2024 + CB 2024 + EU PSD2-SCA-RTS mandates (the canonical EU-PSD2 mandate requires 3DS2-or-equivalent-SCA-for-EU-customer-orders-€30+). Auto-trigger 3DS2 challenge for orders >$200 OR new-customer OR international OR score > 50. Visa-AVN verifies billing-name-matches-card-name-AND-address-matches-card-issuer-data. Dynamic-step-up-authentication: low-risk-order = frictionless 3DS2 (no-challenge); medium-risk = light-step-up (3DS2 challenge + biometric-prompt); high-risk = heavy-step-up (3DS2 challenge + ID-verification + selfie-match + manual-review + 24-hour-fulfillment-delay). Without 3DS2, EU-customer orders fail EU-PSD2-SCA-RTS compliance + liability-shift-from-issuer-to-merchant-on-fraud.
   - **Pillar 4 — Chargeback-recovery + representment + CB-Alert-Ethoca-Verifi-Atlas-network (Week 3–4, 8 hr).** Chargeback-recovery is the canonical $20k–$200k Year-1-lift per $1M GMV per MidMetrics 2024 + Chargeflow 2024 + Chargebacks911 2024 + Midigator 2024 benchmarks. The 4-stage-recipe: (Stage 1 pre-dispute deflection — CB-Alert + Ethoca + Verifi + Atlas networks surface the chargeback-to-the-merchant-before-it-becomes-a-formal-chargeback, allowing refund/credit to prevent the chargeback-fee + the chargeback-rate-from-increasing); (Stage 2 automated representment — Chargeflow / MidMetrics / Midigator / Chargebacks911 auto-generate the representment-letter-with-evidence: shipping-tracking + delivery-confirmation + AVS/CVV-match-results + IP-geolocation + device-fingerprint + customer-communication-history + signed-return-policy + 3DS2-authentication-data + AFPP-PSD2-SCA-compliance-data); (Stage 3 win-rate-optimization — the AI-driven win-rate-optimizer learns which evidence-types-correlate-with-wins-by-reason-code-per-issuer-per-vertical-per-cohort, prioritizing the highest-evidence-payload); (Stage 4 expert-representation — Chargebacks911 for $5M+ GMV brands that want full-service-human-expert-dispute-analysts handling every chargeback). Without this layer, the operator is paying $15–$25 per chargeback-fee + losing 60–80% of chargebacks-that-could-have-been-won-with-evidence.
   - **Pillar 5 — Post-purchase-chargeback-deflection + Gorgias-deflection-flow + Klaviyo-chargeback-prevention-flow + Triple-Whale-chargeback-attribution-overlay (Week 4+, ongoing 2 hr/wk).** Post-purchase-chargeback-deflection intercepts the customer BEFORE they file a chargeback-with-their-issuer — the canonical "if the customer can't reach you, they call their bank" anti-pattern. Recipe: Gorgias-deflection-flow fires when (a) chargeback-notification-arrives-from-Stripe-or-Adyen-or-PayPal + (b) customer-message-history-shows-unresolved-ticket; Gorgias auto-sends: "We see you've raised a dispute with your bank — we'd love to make this right. Click here to receive a [refund/replacement/store-credit] without needing the chargeback." Klaviyo-chargeback-prevention-flow fires when (a) customer-purchase-date-≥30-days-ago AND (b) customer-has-not-opened-a-support-ticket AND (c) customer-engagement-score-declining — pro-active "How's your [Product X]?" outreach to surface-issues-before-they-become-chargebacks. Triple-Whale-chargeback-attribution-overlay tracks per-channel-per-cohort-per-vertical chargeback-rate (paid-social-cohort-A might have 1.5% chargeback-rate vs organic-cohort-B at 0.2%) — the data feeds the channel-mix optimization (cut-allocated-spend-to-cohorts-with-1.5% chargeback-rate if profit-after-fraud-losses-is-negative).

## Common pitfalls (15 from real builds)

1. **No fraud-policy documented.** Treating every order the same at the screening-level is the canonical "we trust the engine and react when chargebacks spike" anti-pattern — the fraud-screening-engine will either over-decline (lost-revenue) or under-decline (chargeback-rate-spikes-to-VAMP-threshold) if there's no policy. **Fix:** write the fraud-policy document (auto-approve / auto-decline / manual-review thresholds + 3DS2-orchestration-rule + AVS/CVV-handling + high-risk-vertical-policy + high-risk-cohort-policy). Get risk-team + operations + finance sign-off. Test with a test-fraud-order from a Stripe-test-card — should auto-decline. *Cost of ignoring: $50k–$200k Year-1 lost-revenue-or-chargeback-cost per Signifyd 2024 + Visa VAMP 2024 benchmarks.*

2. **No 3DS2-orchestration deployed.** Not enabling 3DS2 for EU-PSD2-SCA-compliance + liability-shift-from-issuer-to-merchant-on-fraud is the canonical "we got hit with EU-PSD2-fines + liability-shifts" anti-pattern. EU-PSD2-SCA-RTS mandates 3DS2-or-equivalent-SCA-for-EU-customer-orders-€30+. Without 3DS2, the operator pays $5k–$50k/year in EU-PSD2-non-compliance-fines + loses the liability-shift-on-fraud (issuer-pays-instead-of-merchant). **Fix:** wire Stripe Radar 3DS2 OR Signifyd Dynamic-3DS OR Riskified 3DS2. Auto-trigger 3DS2 challenge for orders >$200 OR new-customer OR international OR score > 50. Verify: a EU-customer-ordering-€30+ triggers 3DS2 (canonical EU-PSD2 mandate). *Cost of ignoring: 50–80% higher chargeback-rate-on-EU-orders + EU-PSD2-non-compliance-fines per Visa 2024 + Mastercard 2024 + EU-PSD2-SCA-RTS 2024 benchmarks.*

3. **No chargeback-recovery layer.** Letting chargebacks-stack-up-without-automated-representment is the canonical "we're paying $15–$25 per chargeback-fee + losing 60–80% of chargebacks-that-could-have-been-won-with-evidence" anti-pattern. **Fix:** onboard Chargeflow / MidMetrics / Midigator / Chargebacks911. Wire CB-Alert + Ethoca + Verifi + Atlas networks for pre-dispute-deflection. Configure automated-representment-letter-generation with all 10 evidence-types populated. Verify: a $100-chargeback-with-shipping-tracking + delivery-confirmation + 3DS2-data wins 70%+ vs $100-chargeback-with-shipping-tracking-alone winning 40%+. *Cost of ignoring: $20k–$200k Year-1 lost-recovery per MidMetrics 2024 + Chargeflow 2024 + Chargebacks911 2024 benchmarks.*

4. **Auto-decline-threshold-too-aggressive.** Setting auto-decline-score > 60 (vs canonical > 85) is the canonical "we're blocking good orders" anti-pattern — false-decline-rate-spikes-to-15%+ + lost-revenue + lost-customer-LTV + customer-complaints-to-Gorgias + brand-reputation-damage. **Fix:** set auto-decline-score > 85 (canonical Signifyd 2024 + Stripe Radar 2024 baseline). Test with 100 historical-good-orders — should not be auto-declined. Verify false-decline-rate-dashboard ≤5–8%. *Cost of ignoring: 5–15% of orders false-declined = $50k–$300k Year-1 lost-revenue per Signifyd 2024 + Stripe Radar 2024 benchmarks.*

5. **No per-vertical tuning.** Applying the same fraud-policy to apparel (low-fraud-rate) and jewelry/watches/electronics (high-fraud-rate) is the canonical "we're under-defending the high-risk-catalog" anti-pattern — high-AOV-$500+-SKUs see 2–4% fraud-rate vs 0.5–1.0% baseline. **Fix:** per-vertical-tuning-recipe: apparel/beauty/home = baseline-policy; jewelry/watches/electronics/CBD = tighten-thresholds + add-Veriff-ID-verification-for-orders-$500+ + manual-review-all-orders-$1,000+. Verify in dashboard: jewelry-orders-<$500 auto-approve-per-baseline; jewelry-orders-$500+ manual-review-with-Veriff-ID-verification. *Cost of ignoring: 2–4× higher fraud-rate-on-high-AOV-catalog = $50k–$500k Year-1 lost-revenue-per-$1M-GMV-of-high-AOV-SKUs per Signifyd 2024 + Veriff 2024 benchmarks.*
