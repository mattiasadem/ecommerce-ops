## Common pitfalls (18 from real builds)

1. **Push-launch-without-VAPID-key-generation-and-service-worker-registration.** A brand launches OneSignal / PushEngage web-push without first generating VAPID-keys (Voluntary-Application-Server-Identification) AND registering the service-worker script (typically a `/sw.js` or `/service-worker.js` file in the site root) AND wiring the canonical `Notification.requestPermission()` UX prompt per RFC 8030 + RFC 8291 + Web-Push-Protocol-Encryption 2024 + MDN-Web-Push-2024 + Airship 2024 + OneSignal 2024 benchmarks. Result: web-push silently fails to deliver (VAPID-mismatch-error / service-worker-not-registered-error), web-push-CTR-tanks at 0%, brand abandons web-push after 30-60 days per Airship 2024 + Leanplum 2024 + OneSignal 2024 + PushEngage 2024 benchmarks. **Fix:** generate VAPID-keys FIRST (per Web-Push-Protocol-RFC-8291-2024 benchmarks via `web-push generate-vapid-keys`); register the service-worker script in the site root BEFORE launching any web-push campaign; wire the canonical `Notification.requestPermission()` UX prompt (delayed-prompt-after-30-seconds OR scroll-trigger OR 50%-page-scroll-trigger OR exit-intent-trigger OR 2nd-visit-trigger per Airship 2024 + Leanplum 2024 + OneSignal 2024 + PushEngage 2024 benchmarks — never first-page-load-trigger which produces 1-3% opt-in rate); test web-push end-to-end BEFORE launching any web-push campaign.

2. **Push-launch-without-iOS-Safari-16.4+-web-push-coverage-check.** A brand launches web-push using only FCM-Android + service-worker-registration-without-iOS-Safari-support-and-without-Apple-Push-API-for-Web-integration per Web-Push-iOS-Safari-16.4-2024 + Safari-Push-Notifications-2024 + Apple-Push-API-for-Web-2024 + Airship 2024 + OneSignal 2024 + PushEngage 2024 + Leanplum 2024 benchmarks.
