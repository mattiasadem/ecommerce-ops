# Klaviyo + Postscript migration (Move #5)

> Move #5 is the canonical **email + SMS platform consolidation** move for $100k–$50M GMV Shopify-DTC brands — the move that consolidates Mailchimp / Brevo / Sendlane / HubSpot / ActiveCampaign / Customer.io / Attentive / Iterable into the **Klaviyo (email) + Postscript (SMS)** default substrate, captures the **−20 to −50% ESP+SMS cost compression** + **+10 to +20% email-attributed revenue lift** from segmentation + automation + deliverability + Shopify-native integration at 1–3-week migration timelines, and unlocks the canonical 4 of the 10 top moves (Move #1 cart-abandon + Move #4 welcome + Move #7 SMS welcome + Move #10 AI ad creative + Move #14 lifecycle-flow-library) that ALL assume Klaviyo + Postscript as the substrate per Klaviyo 2024 + Postscript 2024 + Mailchimp 2024 + Brevo 2024 + Sendlane 2024 + Attentive 2024 + Iterable 2024 + Customer.io 2024 + ActiveCampaign 2024 + HubSpot 2024 + Triple Whale 2024 benchmarks. Ships in **4 paths over 1–3 weeks** — Path A Mailchimp (email) → Klaviyo + Postscript (add SMS) **5–7 days** $0–$800/mo at <50k contacts **−30 to −50% email cost** + new SMS revenue stream (DEFAULT for the typical 25k-contact small-mid DTC store at 8:1 default Year-1 ROI from −35% ESP cost compression + +15% email-attributed revenue + +$300-$800/mo new SMS revenue per Klaviyo 2024 + Postscript 2024 + Mailchimp 2024 + Triple Whale 2024 benchmarks) / Path B Klaviyo-only + Attentive → Klaviyo + Postscript (SMS-only cutover) **3–5 days** $100–$400/mo at 1k–20k SMS subscribers **−40 to −60% SMS cost** / Path C Sendlane / Brevo / HubSpot / ActiveCampaign / Customer.io → Klaviyo + Postscript (full cutover from non-Mailchimp source) **7–10 days** $45–$600/mo at 1k–50k contacts **−20 to −40% total ESP+SMS cost** / Path D Klaviyo-only → Klaviyo + Postscript (add SMS only) **2–3 days** $100–$400/mo **+$200-$800/mo new SMS cost offset by 5–15% revenue lift**. Gated on Shopify store live + Klaviyo paid-tier ($45+/mo) + Postscript Starter/Growth tier ($0–$400/mo) + Shopify-Klaviyo-integration-Wired with Placed-Order + Cart-Started + Checkout-Started events flowing + Shopify-Postscript-integration-Wired + sender-domain-authenticated (SPF + DKIM + DMARC) + TCPA-compliant-SMS-opt-in-checkbox on signup form + 48-72h cutover blackout window chosen + source-ESP full-contact+automation+segment+custom-field backup exported + 8–24 hr/wk operator-capacity during migration.

## When to use this skill

**Use this skill when the operator is on any non-canonical email or SMS platform (Mailchimp, Brevo, Sendlane, HubSpot, ActiveCampaign, Customer.io, Attentive, Iterable, Sendinblue, Omnisend, Drip, ConstantContact, HubSpot Marketing Hub, Mailerlite, ConvertKit, Keap) AND has shipped (or is ready to ship) Move #1 (cart-abandon) + Move #4 (welcome) + Move #7 (SMS) + Move #14 (lifecycle-flow-library) on the canonical Klaviyo + Postscript substrate.** Klaviyo + Postscript migration is structurally distinct from Move #1 (cart-abandon-flow) + Move #4 (welcome-series) + Move #7 (SMS-welcome) + Move #19 (SMS-orchestration) — it is the **prerequisite substrate move** that Move #1 + #4 + #7 + #14 + #19 all assume is already in place — and the operator should only initiate it when ALL of the following are true:

- **Shopify-DTC store live + ≥$100k US DTC GMV for 3 consecutive months.** Move #5 is the canonical platform-consolidation move for DTC stores that have outgrown their starter ESP (Mailchimp free + Brevo free + Sendlane starter) and need the deliverability + automation + segmentation depth of Klaviyo + Postscript. Brands below $100k GMV should defer Move #5 until the unit-economics justify the $45+/mo Klaviyo paid tier OR stay on the free Klaviyo + free-tier-of-Postscript-starter ($0 + $49/mo minimum spend). Verify: Shopify Analytics → Reports → Gross-sales-by-month returning ≥$8.3k/mo for 3 consecutive months OR equivalent $100k+ GMV.
- **Currently on Mailchimp / Brevo / Sendlane / Attentive / Iterable / Customer.io / ActiveCampaign / HubSpot / Omnisend / Drip / Sendinblue / Mailerlite / ConvertKit / Keap OR Klaviyo-only-without-SMS.** Move #5 is a consolidation move; it doesn't make sense to migrate from Klaviyo+Postscript to Klaviyo+Postscript (already there). Brands on Klaviyo + a non-Postscript SMS tool (Attentive + Klaviyo / Iterable + Klaviyo / HubSpot Marketing Hub + Klaviyo) should run Path B (SMS-only cutover). Brands without any SMS tool should run Path A (Mailchimp→Klaviyo+Postscript) OR Path D (Klaviyo-only→Klaviyo+Postscript add-SMS). Verify: current-ESP-name + current-SMS-name fields are populated in `playbooks/05-migrate-to-klaviyo-postscript.md` Step 1 audit inventory.
- **Ready to ship Move #1 (cart-abandon) + Move #4 (welcome) + Move #7 (SMS welcome + cart-abandon) within 30 days of cutover.** Move #5 is the substrate move; without the canonical flows shipped on top, the migration has no measurable lift. Brands that migrate to Klaviyo + Postscript but don't ship the canonical flows cap at −35% ESP cost compression alone (still good ROI but missing the +15-20% email-attributed-revenue lift + the +5-15% SMS revenue lift). Verify: Move #1 + Move #4 + Move #7 playbooks reviewed + flow-templates staged in Klaviyo + Postscript drafts before cutover.
- **Klaviyo paid-tier + Postscript Starter/Growth tier paid.** Klaviyo free tier caps at 250 contacts and disables flows; Postscript Starter requires $49/mo minimum spend. The migration cannot happen on free tiers alone (the source-ESP backup + cutover blackout window + deliverability warm-up plan all require the production tiers). Verify: Klaviyo Settings → Account → Billing showing $45+/mo tier; Postscript Settings → Billing showing $49+/mo minimum spend.
- **Shopify-Klaviyo-integration-Wired with Placed-Order + Cart-Started + Checkout-Started + Subscribed-to-Email-Marketing + Fulfilled-Order events all showing ≥1 event in last 7 days.** Without the Shopify-Klaviyo-integration, the abandoned-cart flow cannot trigger (the `Cart Started` event never fires) + the post-purchase cross-sell flow cannot trigger (the `Placed Order` event never fires) + the welcome series cannot trigger (the `Subscribed to Email Marketing` event never fires). Verify: Klaviyo → Analytics → Metrics showing ≥1 of each metric in last 7 days.
- **Shopify-Postscript-integration-Wired with same 5 metrics.** Postscript needs the same Shopify integration to manage sender-name + opt-out + customer-data + order-data for SMS subscriber profile. Verify: Postscript → Integrations → Shopify showing "Connected" + same 5 metrics flowing.
- **Sender-domain authenticated (SPF + DKIM + DMARC).** Without SPF + DKIM + DMARC, Gmail + Yahoo + Outlook route 30–60% of emails to spam per Gmail-Sender-Guidelines 2024 + Yahoo-Bulk-Sender-Guidelines 2024 + Validity 2024 Sender-Score benchmarks. Klaviyo walks you through SPF + DKIM at Settings → Domains; DMARC is configured at the DNS provider (Cloudflare / Route53 / Namecheap). Verify: Klaviyo → Domains showing green-checkmark on SPF + DKIM; MXToolbox DMARC lookup showing `p=reject` or `p=quarantine` policy.
- **TCPA-compliant-SMS-opt-in-checkbox on signup form + popup + checkout.** Existing subscribers without SMS consent CANNOT be added to Postscript; they must re-opt-in. TCPA fines start at $500 per unsolicited message per FCC-TCPA-2024 + CRSTelecom-SMS-Consent-2024 benchmarks. Verify: signup form + popup + checkout all have separate SMS opt-in checkbox with TCPA-compliant language ("By checking this box, you agree to receive recurring automated marketing SMS messages from [Brand] at the phone number provided. Consent is not a condition of purchase. Msg & data rates may apply. Reply STOP to unsubscribe.").
- **Source-ESP API access + full backup exported (CSV of all contacts + all segments + all automations + all custom fields + 12-months campaign history).** Without the backup, rollback is impossible. The backup should be retained for ≥90 days post-cutover. Verify: source-ESP → Settings → Export producing a CSV with all contacts + all custom fields + all engagement history + 12-months campaign data; CSV stored in a versioned location (Google Drive / S3 / local).
- **Cutover blackout window chosen (48–72h window with historically-low-traffic: Tuesday 10am–Thursday 10am is the safe default).** Avoid Black Friday week, BFCM, Mother's Day, Father's Day, Valentine's Day, Cyber Monday, product launches, active promo windows. Verify: Shopify Analytics → Sessions-by-day-of-week + Sessions-by-hour-of-day returning the chosen 48-72h window as the lowest-traffic period of the week.
- **Stakeholder sign-off on cutover date + cost-delta + rollback plan.** If marketing / ops / finance is a different team, written sign-off is required before the 48-72h cutover window opens. Verify: signed approval from each stakeholder (email / Slack thread / Linear ticket) with cutover date + cost-delta projection + rollback-plan-document link.
- **4–8 hr/wk operator-capacity during Phase 1–5 (audit + build-destination + import + cutover + monitor).** Plus 1 hr/wk operator-capacity during Phase 6 sunset (30+ days post-cutover) for the source-ESP safety window + final data-export verification. Verify: operator has committed 4–8 contiguous hours per week for 1–3 weeks depending on path (A: 1 week, B: 3–5 days, C: 7–10 days, D: 2–3 days).

**Don't use this skill when:**

- **<$100k US DTC GMV OR <1k contacts OR no Shopify store.** Below these thresholds, the $45+/mo Klaviyo paid-tier + $49+/mo Postscript minimum spend cannot be justified by the −20 to −50% ESP cost compression alone; defer until GMV or contact-list grows to justify the migration.
- **Already on Klaviyo + Postscript (the canonical substrate).** No migration needed; ship Move #1 + #4 + #7 + #14 directly on the existing substrate. The canonical Klaviyo + Postscript substrate is THE default; migrating away from it (e.g., to Brevo or Iterable) is not advised per Klaviyo 2024 + Triple Whale 2024 + Shopify 2024 benchmarks.
- **Active BFCM / Black Friday / Mother's Day / Father's Day / Valentine's Day / Cyber Monday / product-launch / active-promo window.** Migration during peak-traffic windows risks 30–60% of cart-abandon emails going to spam + SMS subscribers receiving unsubscribed content; defer until the peak-traffic window passes.
- **No sender-domain authentication possible (no DNS access / no IT support / no Cloudflare / no Route53 / no Namecheap).** Without SPF + DKIM + DMARC, the migration fails deliverability requirements and 30–60% of emails land in spam on Day 1 per Gmail-Sender-Guidelines 2024 + Yahoo-Bulk-Sender-Guidelines 2024; defer until DNS access is provisioned.
- **No TCPA-compliant-SMS-opt-in-checkbox + no ability to add one to signup / popup / checkout.** TCPA fines start at $500 per unsolicited message; migrating to Postscript without the SMS opt-in checkbox creates compliance liability per FCC-TCPA-2024 + CRSTelecom-SMS-Consent-2024; defer until the SMS opt-in checkbox is added to all signup / popup / checkout surfaces.
- **Already-migrated-from-Mailchimp-to-Klaviyo-with-Attentive (Path B-specific).** If on Klaviyo + Attentive, run Path B (SMS-only cutover) — do NOT do a full cutover back to Klaviyo + Postscript from scratch. Path B is the canonical 3–5 day SMS-only cutover for this exact scenario.

## The build (4 paths, 1–3 weeks depending on path)

### Path A — Mailchimp → Klaviyo + Postscript (DEFAULT, 5–7 days)

**Phase 1 — Audit & inventory (Day 1–2):**

1. Export a full contact list from Mailchimp → Account → Settings → Export → CSV. Include all custom fields, all consent timestamps, all engagement history (opens/clicks last 90 days). Verify CSV has ≥99% of source-contact-count.
2. List every active automation/flow on Mailchimp (typically 5–10: welcome, cart-abandon, post-purchase, browse-abandon, winback, sunset, birthday, anniversary, loyalty-tier, NPS-detractor). For each, screenshot the trigger + action steps + record conversion-rate-last-90-days.
3. List every active Mailchimp segment (typically 5–20: All subscribers / Customers / Active-30d / VIP-LTV-top-20% / SMS-consented / Cart-abandon-7d / Winback-90d / Lapsed-180d / etc.). For each, record definition (rule list) + size (count) + use (which campaigns reference it).
4. List every custom field on Mailchimp (typically 5–20: FNAME / LNAME / PHONE / CITY / STATE / COUNTRY / ZIP / SKIN_TYPE / FAVORITE_CATEGORY / FIRST_PURCHASE_DATE / LOYALTY_TIER / etc.). Klaviyo auto-maps FNAME / LNAME / PHONE / CITY / STATE / COUNTRY / ZIP; non-standard fields need manual mapping in Klaviyo → Settings → Custom Properties.
5. Export all Mailchimp campaign history (last 12 months minimum). Even if you don't re-send, you want the engagement data for benchmarking.
6. Measure source-ESP inbox placement BEFORE cutover. Use Google Postmaster Tools (free, requires DNS-verified-domain) + Microsoft SNDS (free, requires IP-signup) + Mail Tester (free, sends test email to scored-inbox) + Validity Sender Score (free) + Litmus Inbox Placement (paid, $99+/mo) to get baseline open-rate + spam-rate + domain-reputation. Record as baseline.
7. **Decision gate before Phase 2:** if your Mailchimp has >10 active automations or >50 active segments, this migration needs ≥2 weeks not 1 week. Do NOT start a BFCM-window cutover.

**Phase 2 — Build the Klaviyo destination (Day 2–4, BEFORE importing contacts):**

1. Sign into Klaviyo (use the free tier initially; upgrade to paid before go-live). Connect Shopify (Settings → Integrations → Shopify) — this is what brings in the `Placed Order` + `Ordered Product` + `Fulfilled Order` + `Subscribed to Email Marketing` + `Cart Started` + `Checkout Started` metrics. Verify each metric shows ≥1 event in last 7 days.
2. Configure sender domain (Settings → Domains → Add sending domain). Klaviyo walks you through SPF + DKIM. DMARC is configured at your DNS provider (Cloudflare / Route53 / Namecheap) — add a TXT record at `_dmarc.yourdomain.com` with value `v=DMARC1; p=quarantine; rua=mailto:dmarc@yourdomain.com` (use `p=reject` for stricter policy). Without all three, expect 30–60% of emails to land in spam on day 1 per Gmail + Yahoo bulk-sender guidelines.
3. Map custom fields (Settings → Custom Properties → Add). For every custom field from Mailchimp, create the Klaviyo equivalent with UPPERCASE_SNAKE_CASE naming. Klaviyo reserves certain names (`email`, `first_name`, `last_name`, `phone_number`, `city`, `region`, `country`, `zip`) — these are auto-mapped.
4. Build the 6 core flows in this priority order. Every flow should be REVIEWED + TESTED before cutover:
   - **Flow 1: Welcome series (5 emails + 1 SMS)** — see Move #4 playbook for full build. Use the Klaviyo Welcome Series template as starting point; replace copy with brand voice; suppress for existing customers (`Placed Order ≥ 1 times`).
   - **Flow 2: Abandoned cart (3 emails + 2 SMS)** — see Move #1 playbook for full build. Trigger: `Checkout Started zero times within the flow window`. Use unique 10% off code on Email 3, sent at 24h. Suppress if customer has placed order in last 7 days.
   - **Flow 3: Post-purchase upsell confirmation + cross-sell** — see Move #2 playbook for full build. Trigger: `Placed Order`. Send 7 days after delivery. Use ReConvert for Shopify-native upsell or build a 2-email confirmation + review-request flow in Klaviyo.
   - **Flow 4: Browse abandonment (1 email)** — trigger: `Viewed Product ≥ 1 time AND zero Placed Order in last 24h`. Lower priority; ship 30 days after first 3 flows are live.
   - **Flow 5: Customer winback (90-day)** — trigger: `last purchase ≥ 90 days ago AND last email engagement ≥ 30 days ago`. 3 emails over 14 days, 15% off.
   - **Flow 6: Sunset (suppress non-engagers)** — trigger: `zero opens in last 180 days`. Move to a "low-engagement" segment; Klaviyo auto-suppresses for inbox providers if you enable the sunset flow.
5. Create the 5 core segments before importing:
   - `All subscribers` (newsletter list)
   - `Customers` (placed ≥1 order)
   - `Active subscribers (last 90d)` (engagement-based; protects deliverability)
   - `VIP customers` (LTV ≥ top 20% — calculate from your store data)
   - `SMS-consented` (sms_consent = true) — separate segment for SMS campaigns
6. Create signup forms (Klaviyo → Signup Forms → Popups). Replace Mailchimp popups with Klaviyo equivalents before cutover. Set popup to fire on right URL (homepage, product page, exit-intent). Use same brand colors + copy. **Critical: include a separate SMS opt-in checkbox** in the Klaviyo form with TCPA-compliant language.

**Phase 3 — Import contacts + suppress source ESP (Day 4–5):**

1. Use Klaviyo's 1-click Mailchimp importer (Settings → Integrations → Mailchimp → Import). Verify the import: `SELECT COUNT(*) FROM Klaviyo; profiles` should match source-count ±1% margin. Spot-check 10 random profiles for custom-field fidelity.
2. Configure suppression: import Mailchimp's unsubscribe-list + bounce-list + complaint-list into Klaviyo's suppression-list. This prevents the post-cutover spam-complaint spike from old Mailchimp-unsubscribers.
3. Tag imported profiles with `mailchimp_import_2026_07_11` (use today's date) to enable cohort-tracking for the 30-day parallel-run.
4. Send a "We've upgraded our email platform" email to all imported subscribers (Mailchimp-side, one last send) explaining the migration + the SMS opt-in invitation + the "Add SMS for early access" CTA linking to a dedicated signup page.

**Phase 4 — Stand up Postscript + add SMS flows (Day 5–6):**

1. Sign into Postscript (Starter tier $0 + $49/mo minimum, OR Growth $100/mo). Connect Shopify (Settings → Integrations → Shopify).
2. Connect Postscript to Klaviyo (Postscript → Integrations → Klaviyo). This syncs the SMS subscriber list + consent timestamps + opt-in source between the two platforms.
3. Build the 4 core SMS flows in this priority order:
   - **SMS 1: Welcome SMS** — send immediately on signup; 1 SMS at signup with brand voice + opt-out instructions.
   - **SMS 2: Cart Abandon 1** — send at 1h after cart-start; 1 SMS with direct cart-link + brand voice.
   - **SMS 3: Cart Abandon 2 (escalation with discount)** — send at 24h after cart-start if cart not recovered; 1 SMS with 10% off code.
   - **SMS 4: Post-Purchase Review Request** — send at Day 7 post-delivery; 1 SMS with review-link + brand voice.
4. Import existing SMS-consented subscribers from Klaviyo → Postscript via the Klaviyo-Postscript integration. Verify the import: `SELECT COUNT(*) FROM Postscript; subscribers` should match Klaviyo-SMS-consented-count ±1% margin.
5. For email-only subscribers (no SMS consent), create a dedicated re-opt-in campaign: SMS opt-in email at Day 7 post-cutover with "Add SMS for early access + free shipping" CTA. Target: 10–25% of email-only subscribers add SMS.

**Phase 5 — Cutover + monitor (Day 6–7, the 48–72h window):**

1. **Day 6 morning:** Pause all Mailchimp campaigns + automations (set status to "Paused" not "Deleted" — rollback option). Verify Mailchimp is paused but data is intact.
2. **Day 6 afternoon:** Activate all Klaviyo flows + activate all Postscript SMS flows. Test end-to-end with a sandbox email + sandbox phone number: signup → welcome email → cart-start → cart-abandon email + SMS → checkout → order-confirmation → post-purchase review-SMS at Day 7 (simulated).

4. **Day 7 evening:** If all metrics are in green band, KEEP Mailchimp paused (do NOT delete). Source-ESP-30-day-parallel-run is the safety net.
5. **Day 14 + Day 30:** Re-check inbox-placement via Google-Postmaster-Tools + Microsoft-SNDS + Mail-Tester. Compare to Day 1 baseline. Spam-complaint-rate should be ≤0.1% (vs Mailchimp baseline).

**Phase 6 — Sunset source ESP (Day 30+, after successful cutover):**

3. Document the migration in a runbook (`/docs/migration-runbooks/2026-07-11-mailchimp-to-klaviyo-postscript.md`) with cutover-date + cost-delta + revenue-lift + lessons-learned.

## Common pitfalls (15 from real builds)

1. **Migrating during peak-traffic window (BFCM / Black Friday / Mother's Day / Father's Day / Valentine's Day / Cyber Monday / product-launch / active-promo)** — Cutover during peak-traffic windows risks 30–60% of cart-abandon emails going to spam + SMS subscribers receiving unsubscribed content; deliverability-reputation damage takes 30–90 days to repair. **Fix:** Choose a 48–72h cutover window with historically-low-traffic (Tue 10am–Thu 10am is the safe default); avoid all peak-traffic windows; reschedule if any peak-traffic event lands within 14 days of planned cutover.
2. **Importing contacts BEFORE building the Klaviyo destination flows** — The trap: import 25k contacts first, then realize the welcome flow has a broken template, and now you've been emailing 25k people with a broken-template + unsubscribe-spike + spam-complaint-spike for 4 hours. **Fix:** Build Phase 2 (Klaviyo destination) BEFORE Phase 3 (import contacts); build all 6 core flows + 5 core segments + signup forms first; verify each flow end-to-end with a sandbox profile before importing.
3. **Missing SPF / DKIM / DMARC sender-domain authentication** — Without all three, Gmail + Yahoo + Outlook route 30–60% of emails to spam per Gmail-Sender-Guidelines 2024 + Yahoo-Bulk-Sender-Guidelines 2024 + Validity-2024-Sender-Score benchmarks. **Fix:** Configure SPF (Klaviyo auto-walks-you-through) + DKIM (Klaviyo auto-walks-you-through) + DMARC (add TXT record at `_dmarc.yourdomain.com` with `v=DMARC1; p=quarantine; rua=mailto:dmarc@yourdomain.com`); verify all 3 records green-checkmark in Klaviyo → Domains + MXToolbox DMARC lookup.
4. **Migrating to Postscript without TCPA-compliant-SMS-opt-in-checkbox** — Existing subscribers without SMS consent CANNOT be added to Postscript; they must re-opt-in. TCPA fines start at $500 per unsolicited message per FCC-TCPA-2024 + CRSTelecom-SMS-Consent-2024. **Fix:** Add separate SMS opt-in checkbox to ALL signup / popup / checkout surfaces BEFORE Postscript-migration; checkbox must be un-checked-by-default + separate from email-opt-in + explicit TCPA-consent-language ("By checking this box, you agree to receive recurring automated marketing SMS messages from [Brand] at the phone number provided. Consent is not a condition of purchase. Msg & data rates may apply. Reply STOP to unsubscribe.").
5. **Pre-checked SMS opt-in checkbox (TCPA violation)** — TCPA + FCC explicitly prohibit pre-checked consent checkboxes; consent must be affirmative (the user must actively check the box, not un-check a default-checked box). Pre-checked checkbox = $500 per unsolicited message fine. **Fix:** Verify the SMS opt-in checkbox is unchecked-by-default on signup/popup/checkout; add JavaScript to force-uncheck on page-load.
6. **Bundling SMS opt-in with email opt-in (no express consent)** — TCPA requires separate, express consent for SMS; bundling SMS-opt-in with email-opt-in (single checkbox that says "I agree to receive email AND SMS marketing") does NOT meet the express-consent requirement. **Fix:** Add separate SMS opt-in checkbox; do NOT bundle with email opt-in; ensure the SMS consent language is explicit ("I agree to receive SMS marketing from [Brand]").
7. **Sending full-volume on cutover-day (cold-IP warm-up miss)** — Klaviyo's IP reputation starts cold; sending 25k+ emails on Day 1 from a cold IP triggers 30–60% spam-folder-rate per Validity-2024-Sender-Score benchmarks. **Fix:** Implement the canonical deliverability-warm-up-plan: Week 1: 25% of source-ESP send volume (most-engaged 25% of subscribers); Week 2: 50%; Week 3: 75%; Week 4: 100% (gradual IP-warm-up at Klaviyo + gradual-list-reactivation); keep Mailchimp paused (not deleted) as the safety net for rollback.
8. **Custom-field mapping errors (5–15% of records lose custom-field fidelity)** — Klaviyo auto-maps standard fields (FNAME / LNAME / PHONE / CITY / STATE / COUNTRY / ZIP) but non-standard fields (FIRST_PURCHASE_DATE / SKIN_TYPE / FAVORITE_CATEGORY / LOYALTY_TIER) require manual mapping. Manual mapping has 5–15% error rate (typos in field-name + field-type mismatches + date-format-mismatches). **Fix:** Use UPPERCASE_SNAKE_CASE naming convention; create a custom-field-mapping-document (CSV with source-field-name + Klaviyo-field-name + field-type + sample-value); spot-check 10 random profiles per custom-field after import.
9. **Dropping source-ESP automations instead of migrating them 1:1** — Path A Mailchimp has 6–10 active automations typically; brands that migrate only the top-3 (cart-abandon + welcome + post-purchase) lose the lower-priority flows (browse-abandon + winback + sunset + birthday + anniversary + loyalty-tier + NPS-detractor) which collectively drive 30–50% of structural upside per Klaviyo-Flow-Library-2024 + Move #14-lifecycle-flow-library benchmarks. **Fix:** Migrate ALL source-ESP automations 1:1 to Klaviyo with full email-content + trigger-config + suppression-rules preserved; prioritize by monthly-revenue-contribution (each automation's last-90-day-revenue from source-analytics).
10. **No source-ESP backup exported before cutover** — Without the backup (full contact CSV + all segments + all automations + 12-months campaign-history), rollback is impossible if cutover fails. **Fix:** Export full source-ESP backup at Phase 1 Step 1; retain for ≥90 days post-cutover in a versioned location (Google Drive + S3 + local).
11. **Deleting source-ESP data on Day 1 (no rollback option)** — Brands that delete Mailchimp data on Day 1 of cutover have no rollback option if Klaviyo deliverability fails. **Fix:** Pause source-ESP (status = Paused, not Deleted) for the 30-day-parallel-run window; cancel subscription only after Day-30 metrics are in great band; retain data backup for ≥90 days post-cancellation.
12. **Re-using Mailchimp flow-templates in Klaviyo without re-coding** — Mailchimp's drag-and-drop template editor produces different HTML markup than Klaviyo's block-based template editor; copy-pasting Mailchimp HTML into Klaviyo loses mobile-responsiveness + brand-styling in 10–30% of templates. **Fix:** Re-build each flow-template in Klaviyo's block-based template editor from scratch; use Klaviyo's drag-and-drop blocks (Text + Image + Button + Columns + Product + Divider) for mobile-responsive + brand-styled rendering.
13. **No SMS opt-in re-consent campaign for email-only subscribers** — Email-only subscribers (no SMS consent) cannot be auto-added to Postscript; without a re-opt-in campaign, the SMS subscriber list starts at 0–25% of total-list-size vs the 100% potential. **Fix:** Send a dedicated "Add SMS for early access + free shipping" re-opt-in email at Day 7 post-cutover with a single CTA linking to a dedicated SMS signup page; target: 10–25% of email-only subscribers add SMS.
14. **Migrating to Postscript without verifying Shopify-Postscript-integration is wired** — Postscript needs the Shopify integration to manage sender-name + opt-out + customer-data + order-data for SMS subscriber profile. Without the integration, SMS subscriber profiles are missing order-history (relevant for cart-abandon SMS) + customer-LTV (relevant for VIP-SMS segmentation) + opt-out-state (TCPA-liability). **Fix:** Verify Shopify-Postscript-integration BEFORE cutover (Postscript → Integrations → Shopify showing "Connected" + same 5 metrics flowing as the Klaviyo-Shopify-integration).
15. **Treating Move #5 as a one-time task instead of an ongoing substrate optimization** — Klaviyo + Postscript is a substrate, not a destination. The canonical substrate requires ongoing optimization: weekly Klaviyo flow-analytics review (open-rate + CTR + CTOR + unsubscribe-rate + revenue-per-$1-sent per flow) + monthly Postscript SMS-deliverability review (DLR + opt-out-rate + click-rate + spam-complaint-rate) + quarterly deliverability-audit (Google-Postmaster-Tools + Microsoft-SNDS + Validity-Sender-Score + Litmus-Inbox-Placement) + annual Klaviyo-segment-refresh (RFM-segment refresh + cohort-LTV-overlay + AI-segment-Wire for Move #20 generative-AI-engine). **Fix:** Schedule the canonical 4-cadence (weekly flow-analytics + monthly SMS-deliverability + quarterly deliverability-audit + annual Klaviyo-segment-refresh); document in runbook; assign on-call rotation; instrument baseline-vs-current comparison to detect regressions.
