### Operator anti-targets

Do not use this skill if any of these is true:

- the merchant has <1,000 contacts/mo across all channels and <$100k/yr in contact-center cost — the WFM / QM / WFO / STT / voice-analytics stack doesn't pay back at <$100k/yr;
- the merchant has no documented channel matrix, no documented per-tier-routing rule, no documented per-vertical-specialty-routing rule, no documented QA-rubric, no documented WFM-staffing-roster, and no documented Triple Whale / Polar / Northbeam attribution layer;
- the merchant wants a contact-center program but is unwilling to invest the 6–12 hr/wk operator time, the 8–15 hr/wk supervisor time, the 12–25 hr/wk agent-time, the 5–10 hr/wk QA-time, and the 5–10 hr/wk WFM-time that a real omnichannel contact-center program requires;
- the merchant is selling high-touch / made-to-order / personalized / perishable / hazmat / regulated goods where the customer expects voice-only high-touch service and the AI-CS deflection would damage brand-equity;
- the merchant is selling counterfeit-prone / high-counterfeit-risk categories (per Move #58 brand-protection-counterfeit-enforcement) and has no authentication-orchestration capability — contact-center without authentication in counterfeit-prone categories creates a parallel fraud-channel risk.

## Common pitfalls (18 from real builds)

1. **Wiring IVR before defining per-tier-routing.** IVR-flow branches to the wrong tier because the routing rule isn't documented. **Fix:** define the per-tier-routing rule (tier1 / tier2 / tier3 / tier4 / specialty / VIP / loyalty / B2B / trade-show / premium-luxury) FIRST, then wire IVR-deflection-to-AI-CS-agent / IVR-deflection-to-human-agent / IVR-outbound-call-back to the documented rule.

2. **WFM without per-channel-multiplier.** Voice forecasting is treated as the canonical channel-multiplier and chat / email / SMS / WhatsApp / Messenger / IG DM / TikTok DM are staffed at voice-equivalent. **Fix:** wire WFM-channel-multiplier per channel — chat-multiplier is typically 1.5–2.5× voice-multiplier per ICMI 2026 + SSA 2026 + Talkdesk 2026 + Five9 2026 + NICE 2026 benchmarks; email-multiplier is 0.6–1.2×; SMS / WhatsApp / Messenger / IG DM / TikTok DM multiplier is 1.0–1.5×.

3. **QA-coverage-target below 10%.** QA-rubric samples 2–5% of contacts and misses systemic issues. **Fix:** set QA-coverage-target at 10% minimum (random sampling), 25% (risk-based), 50% (LTV-cohort-based), 100% (regulatory / compliance-phrase / PCI / PII / TCPA / TSR / FCC / state-call-recording).

4. **AI-deflection-rate without handoff-context-loss-rate.** AI-CS deflection saves $8–$22/deflection but customers re-explain themselves when escalated, eroding the savings and CSAT. **Fix:** wire AI-CS-handoff-context-loss-rate — every handoff must include the AI-CS-conversation-thread, the customer-record-cohort-LTV, the per-channel-previous-touch, and the AI-CS-summary. Drift fails CI.

5. **STT-engine single-language.** Whisper / Deepgram / AssemblyAI / AWS Transcribe / Azure Speech / Google STT / Speechmatics / Symbl.ai default to English-only and miss multilingual-coverage. **Fix:** wire STT-engine per-language per-vertical — English (WER 2–4%), Spanish (3–5%), French (3–5%), German (3–5%), Italian (3–5%), Portuguese (3–5%), Japanese (5–8%), Korean (5–8%), Chinese (5–8%), Hindi (8–12%), Arabic (8–12%). Track STT-accuracy-target per-language.

6. **No PCI-redaction on STT / recording.** Voice recordings and STT transcripts leak card numbers / CVV / expiration to QA agents / supervisors / WFM / engineering. **Fix:** wire PCI-redaction at the STT-layer (Deepgram + AssemblyAI + AWS Transcribe + Azure Speech + Google STT all support PCI-redaction) AND at the recording-layer (Observe.AI + Cradle + NICE + Playvox + Calabrio + Quality 360 + Gong + Chorus + CallRail + Invoca + Marchex all support PCI-redaction). Drift fails PCI-DSS / PCI-PAL audit.

7. **No PII-redaction on STT / recording.** Voice recordings leak SSN / DOB / address / phone / email to QA agents / supervisors / WFM / engineering. **Fix:** wire PII-redaction at the STT-layer AND at the recording-layer. Drift fails GDPR / UK ICO / UK PECR / Canada PIPEDA / Australia Privacy Act / Japan PIPA / Brazil LGPD / Singapore PDPA / Hong Kong PDPO / South Korea PIPA / NZ Privacy Act / India Privacy / CCPA / CPRA / VCDPA / CPA / CTDPA / UCPA / Colorado Privacy Act audit.

8. **No DNC-scrub before outbound-call.** Outbound-call is dialed against numbers on the DNC-list and triggers TCPA / TSR / FCC / state-call-recording / FCC-robocall violations. **Fix:** wire DNC-scrub against the national DNC-list AND state-level DNC-list AND internal-opt-out-list AND FCC-robocall-rules (the 2024 FCC rule that requires prior-express-consent for AI-voice-calls). Drift triggers $500–$1,500/violation.

9. **No STIR-SHAKEN on outbound-call.** Outbound-call is missing STIR-SHAKEN attestation and is flagged as spam-robocall. **Fix:** wire STIR-SHAKEN attestation per FCC rule (Attestation A / B / C) for every outbound-call. Drift fails FCC-robocall / carrier-spam-flag / call-labeling.

10. **No e911 / emergency-call-routing.** Voice calls cannot reach 911 / emergency-services or route to the wrong PSAP. **Fix:** wire e911 / emergency-call-routing (Twilio / Bandwidth / Vonage / Agora + NICE / Five9 / Talkdesk / Genesys / Avaya / Amazon Connect / Google CCAI / Microsoft Dynamics 365 / Cisco Webex / Zoom Contact Center all support e911 / KEL / location-services). Drift triggers FCC / state-PUCO / state-CPUC violations.

11. **Per-tier-routing without skill-based-routing-rule.** Contacts route to a tier but the agent-skill-set doesn't match the contact-need. **Fix:** wire skill-based-routing-rule — per-tier-skill-requirement (vertical / language / channel / cohort / LTV / stage / system-access) with documented per-vertical-specialty (apparel / beauty / food / pet / home / kids / gift / B2B / trade-show / premium-luxury / electronics / supplements) and documented per-language-coverage (English / Spanish / French / German / Italian / Portuguese / Japanese / Korean / Chinese / Hindi / Arabic). Drift fails FCR.

12. **No cohort-LTV-overlay on contact-center-engaged buyers.** Every contact-center-engaged buyer is attributed as a generic visitor and the recovered-revenue is invisible. **Fix:** wire cohort-tag (contact_center_engaged / omnichannel_engaged / AI_deflected / human_handled / voice_engaged / chat_engaged / email_engaged / sms_engaged / whatsapp_engaged / messenger_engaged / instagram_dm_engaged / tiktok_dm_engaged / in_app_engaged / video_engaged / screen_share_engaged) via Triple Whale / Polar / Northbeam cohort-LTV-overlay. Drift fails per-channel-PnL attribution.

13. **Per-channel-PnL missing the cost-side.** Per-channel recovered-revenue is tracked but the per-channel cost-per-contact is invisible. **Fix:** wire per-channel cost-per-contact (voice / chat / email / SMS / WhatsApp / Messenger / Instagram DM / TikTok DM / in-app / video / screen-share) with documented per-FTE-cost + per-FTE-handle-time + per-FTE-AHT + per-FTE-occupancy + per-FTE-overhead. Drift fails contribution-margin math.

14. **No AI-CS-fallback-rate tracking.** AI-CS deflection fails silently (the customer abandons the chat, opens a new chat, the AI doesn't recognize the previous context). **Fix:** wire AI-CS-fallback-rate (AI-fails / AI-deflected) with documented AI-CS-tone-mismatch-rate and AI-CS-multilingual-coverage. Drift fails AI-CS-CSAT and erodes the AI-CS-cost-savings.

15. **No AI-QA-auto-scoring-accuracy validation.** AI-QA auto-scores 70–95% of contacts but the precision / recall / F1 vs human-calibration is invisible. **Fix:** wire monthly calibration cycle with documented AI-QA-precision / AI-QA-recall / AI-QA-F1. Drift fails QA-coverage-target and erodes the AI-QA-cost-savings.

16. **No per-vertical-specialty routing in B2B / trade-show / premium-luxury verticals.** B2B / trade-show / premium-luxury contacts route to tier1 (general) and miss the per-vertical-specialty-routing-rule. **Fix:** wire per-vertical-specialty-tier1-tier2-tier3-tier4 routing matrix — B2B requires tier3 with trade-show-specialty skill; premium-luxury requires tier3 with luxury-specialty skill; trade-show requires tier3 with trade-show-specialty skill; gift requires tier2 with gift-specialty skill. Drift fails FCR + CSAT + per-vertical-PnL.

17. **No two-party-consent disclosure in California / Florida / Illinois / Pennsylvania / Washington / Massachusetts / Maryland / New York.** Voice recordings are made without two-party-consent disclosure and trigger state-call-recording violations. **Fix:** wire two-party-consent disclosure in the IVR-flow + AI-CS-opening-message + email-channel + chat-opening-message — the disclosure must be clear, conspicuous, and EARLY in the conversation. Drift triggers $100–$500/recording + class-action exposure.

18. **No per-cohort-monthly-quality-scorecard.** Per-channel-metrics (ASA / AHT / FCR / occupancy / schedule-adherence / abandon-rate / service-level / forecast-accuracy / CSAT / NPS / AI-deflection-rate / AI-CS-CSAT / AI-CS-FCR / AI-CS-AHT / AI-CS-resolution-rate / AI-CS-handoff-rate / AI-CS-fallback-rate / AI-QA-auto-scoring-coverage / AI-QA-precision / AI-QA-recall / QA-coverage / STT-accuracy / STT-cost-per-minute / voice-analytics-coverage / per-channel cost-per-contact / per-channel recovered-revenue / per-channel contribution-margin / cohort-LTV) are tracked in isolation. **Fix:** wire per-cohort-monthly-quality-scorecard that surfaces ALL canonical metrics above in one view, with documented per-tier / per-vertical / per-channel / per-cohort / per-language / per-LTV breakdown. Drift fails the canonical contact-center-engine.
