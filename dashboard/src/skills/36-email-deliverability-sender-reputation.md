## What "best in class" looks like

A best-in-class email-deliverability engine has all 7 of these properties:

1. **All 7 authentication records at the DNS layer** — SPF (`v=spf1 include:klaviyo.com ~all` or similar) + DKIM (Klaviyo auto-walks-you-through, returns 2 CNAMEs) + DMARC (`v=DMARC1; p=reject; rua=mailto:dmarc@yourdomain.com; pct=100; adkim=s; aspf=s`) + BIMI (`v=BIMI1; l=https://yourdomain.com/bimi-logo.svg; a=https://yourdomain.com/bimi-vmc.pem`) + ARC (`v=ARC1; ...`) + MTA-STS (`v=STSv1; id=...`) + TLS-RPT (`v=TLSRPTv1; rua=mailto:tlsrpt@yourdomain.com`). Without all 7, Gmail + Yahoo + Microsoft apply the 2024 bulk-sender-mandate enforcement that hard-routes to spam (the canonical "we lost 30% of email revenue overnight" anti-pattern per Gmail Sender Guidelines 2024 + Yahoo Bulk Sender Guidelines 2024 + Microsoft SNDS 2024 + Validity 2024 + Return Path 2024 benchmarks).

2. **All 4 reputation-monitoring sources wired** — Google Postmaster Tools (free, surfaces domain-reputation "High/Medium/Low" + IP-reputation + spam-rate + inbox-rate + DMARC-alignment-rate + encryption-rate for Gmail traffic) + Microsoft SNDS (free, surfaces IP-reputation + junk-rate + complaint-rate + trap-hit-count + spam-complaint-rate for Outlook/Hotmail/Live traffic) + Validity Sender Score (free, surfaces 0-100 IP-reputation-score + domain-reputation-score for the major mailbox providers) + Litmus Inbox Placement (paid $99+/mo, surfaces per-mailbox-provider inbox-vs-spam-vs-promotions-vs-primary-vs-updates-tab routing for Gmail + Outlook + Yahoo + Apple Mail + HEY + AOL + Comcast + Cox + Charter + mobile-mail-app + dark-mode-mail-app variants). The canonical weekly cadence: open Postmaster Tools + SNDS + Sender Score every Monday morning + Litmus Inbox Placement once per month + Gmail Feedback Loop + Microsoft JMRP + Yahoo Feedback Loop once per week.

3. **List-hygiene discipline ≥95% clean rate** — maintain suppression-list (unsubscribes + bounces + complaints + manual-removals) with double-validation (synced from Klaviyo's auto-suppression + from Google Feedback Loop + from Microsoft JMRP + from Yahoo Feedback Loop + from Litmus spam-test results + from manual-CSV-upload-from-help-desk) + run sunset-policy-90-day-no-open (auto-tag + auto-remove at Day 120 if no re-engagement) + run 3-email winback-campaign at Day 90 (target: 5–15% conversion-back-to-engaged per Klaviyo 2024 + Omnisend 2026 + Sailthru 2024 + Iterable 2024 benchmarks). The canonical "list-decay-rate < list-growth-rate" check: if your list is growing 5%/mo but your engagement-rate is dropping 2%/mo, you have a list-hygiene problem that compounds over 6–12 months into inbox-placement collapse.

4. **IP-warmup playbook documented + practiced** — if on shared IP (Klaviyo + Postscript default), monitor the shared-pool reputation weekly (Klaviyo exposes a "shared-pool reputation score" in the deliverability dashboard) + have an exit-strategy to dedicated-IP if the shared-pool degrades (Klaviyo dedicated IP $200/mo + Postscript dedicated IP included). If on dedicated IP, run the canonical 4-week 25%-50%-75%-100% warmup cadence before any volume increase (the canonical "we sent 50k emails from a cold IP and got blacklisted by Spamhaus within 4 hours" anti-pattern per Klaviyo IP Warmup 2024 + Validity 2024 + Postmark 2024 + SendGrid 2024 + Mailgun 2024 benchmarks). If blacklisted, follow the delisting-playbook for Spamhaus (DBL + SBL + XBL + ZEN) + Spamcop + Barracuda + Cisco Talos + URI + Reputation Authority + Sender Score — each has a distinct delisting-request-form + delisting-SLA + re-incidence-policy.

5. **Content-engagement flywheel sustained** — open-rate ≥40% + CTR ≥5% + CTOR ≥10% + unsubscribe-rate ≤0.5%/mo + spam-complaint-rate ≤0.02% per Klaviyo 2024 + Postscript 2024 + Omnisend 2026 + Litmus 2024 + Return Path 2024 + Iterable 2024 + Sailthru 2024 benchmarks. Below these thresholds, Gmail + Yahoo + Microsoft deprioritize your sending IP in the inbox-placement algorithm (the canonical "we had a 60% open rate 6 months ago, now we're at 25% — what changed?" answer is usually engagement-flywheel decay per Gmail Tabbed Inbox 2024 + Litmus Spam Filter Testing 2024 + Klaviyo Engagement Score 2024 + Iterable 2024 benchmarks).

### Phase 1 — Authentication-stack hardening (Day 1–2, 6 hr)

The canonical Gmail-Yahoo-Feb-2024-mandatory-policy-compliance-checklist:

1. **SPF record.** Add TXT record at `yourdomain.com` (the root, NOT a subdomain): `v=spf1 include:klaviyo.com include:sendgrid.net include:mailgun.com ~all` (include all ESPs + transactional-senders; start with `~all` for soft-fail during transition, tighten to `-all` after Week 4 of deliverability-confirmed). Verify at MXToolbox SPF lookup (https://mxtoolbox.com/spf.aspx ) — must show the SPF record found + DNS-lookup-count-below-10 (the canonical RFC 7208 limit). Klaviyo auto-provides the include-statement at Settings → Domains.

2. **DKIM records.** Klaviyo auto-walks-you-through: Settings → Domains → Add sending domain → enter `yourdomain.com` → Klaviyo generates 2 CNAMEs (typically `kl1._domainkey.yourdomain.com` + `kl2._domainkey.yourdomain.com`) → add both at your DNS provider → verify-green-checkmark in Klaviyo → Domains. Postscript auto-walks-you-through similarly. Without DKIM, Gmail + Yahoo deliverability-routing-downgrades by 10–20% inbox-placement-rate per Gmail Sender Guidelines 2024 + Yahoo Bulk Sender Guidelines 2024 + Validity 2024 Sender Score benchmarks.

3. **DMARC record (Week 1: `p=none` → Week 2: `p=quarantine pct=10` → Week 4: `p=quarantine pct=100` → Week 8: `p=reject pct=100`).** Add TXT record at `_dmarc.yourdomain.com`: starting with `v=DMARC1; p=none; rua=mailto:dmarc@yourdomain.com; pct=100; adkim=s; aspf=s`. The canonical 8-week-progression: Week 1 (p=none — observe-only), Week 2 (p=quarantine pct=10 — quarantine 10% of failures), Week 4 (p=quarantine pct=100 — quarantine all failures), Week 8 (p=reject pct=100 — reject all failures). Without tightening-to-p=reject, spoofed-emails-from-your-domain land in recipients'-inbox, damaging your sender-reputation per DMARC 2024 + Gmail Sender Guidelines 2024 + Yahoo Bulk Sender Guidelines 2024 + Validity 2024 Sender Score + Return Path 2024 benchmarks.

4. **BIMI record (Path B+, $1,200-$1,500/yr VMC).** Buy a VMC (Verified Mark Certificate) from Entrust or DigiCert ($1,200-$1,500/yr; requires trademark-registration-of-brand-logo + brand-verification-process-of-2-4-weeks). Host the SVG-logo at `https://yourdomain.com/bimi-logo.svg` (must be square SVG, no animations, no embedded-scripts, ≤32KB). Add TXT record at `default._bimi.yourdomain.com`: `v=BIMI1; l=https://yourdomain.com/bimi-logo.svg; a=https://yourdomain.com/bimi-vmc.pem`. BIMI shows the brand-logo in Gmail + Yahoo + Apple Mail + Comcast inboxes (the canonical "blue-checkmark-equivalent-for-email" branding per BIMI 2024 + Gmail Sender Guidelines 2024 + Yahoo Bulk Sender Guidelines 2024 benchmarks).

5. **ARC (Authenticated Received Chain) — auto-handled by Klaviyo + Postscript.** No operator-action-required; ARC preserves the authentication-chain when emails-forward-through-mailing-lists-or-intermediaries. Without ARC, mailing-list-forwarding breaks DKIM (the canonical "we have a customer-forwarding-our-newsletter-to-a-co-worker-via-a-mailing-list-and-our-DKIM-keeps-breaking" anti-pattern per ARC 2024 + DMARC 2024 + Validity 2024 Sender Score benchmarks).

6. **MTA-STS (Mail Transfer Agent Strict Transport Security).** Add TXT record at `_mta-sts.yourdomain.com`: `v=STSv1; id=20240101000000Z` (the ID is a timestamp; bump-on-policy-change). Plus: host `https://mta-sts.yourdomain.com/.well-known/mta-sts.txt` with content `version: STSv1; mode: enforce; mx: mail.yourdomain.com; max_age: 86400`. The canonical "enforce TLS-on-all-inbound-email" recipe per MTA-STS 2024 + Gmail Sender Guidelines 2024 + Yahoo Bulk Sender Guidelines 2024 benchmarks.

7. **TLS-RPT (TLS Reporting).** Add TXT record at `_smtp._tls.yourdomain.com`: `v=TLSRPTv1; rua=mailto:tlsrpt@yourdomain.com`. The canonical "get-a-daily-report-of-TLS-failures-on-inbound-mail" recipe per TLS-RPT 2024 + Gmail Sender Guidelines 2024 + Yahoo Bulk Sender Guidelines 2024 benchmarks.

8. **List-Unsubscribe-Post header (1-click-unsubscribe).** Klaviyo + Postscript + all major ESPs auto-include the `List-Unsubscribe` + `List-Unsubscribe-Post: <mailto:unsubscribe@yourdomain.com>, <url:https://yourdomain.com/unsubscribe>` headers. Verify via Gmail → Open email → Reply dropdown → "Unsubscribe" option-visible (if not, header is missing).
