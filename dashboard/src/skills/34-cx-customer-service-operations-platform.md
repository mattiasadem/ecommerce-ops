## When to use this skill

Use this skill when the operator is doing **≥500 orders/month** at **≥$50 AOV** AND has **≥100 tickets/month** AND/OR is losing $40k+/year to slow-first-response-time + low-FCR + low-CSAT-driven-churn + manual-ticket-triage-cost + missed-chargeback-deflection + missed-upsell-cross-sell. Specific signals: the operator is managing tickets in a shared Gmail-inbox + Google-Sheets + Slack-channel (the canonical "we have 8 people monitoring a single inbox" anti-pattern), first-response-time is >4 hr (the canonical "customers wait 8 hours for a refund status update" anti-pattern per Gorgias 2024 + Help Scout 2024 benchmarks), FCR is <60% (the canonical "we're answering the same shipping-status question 5 times per customer" anti-pattern per Gorgias FCR 2024 + Help Scout FCR 2024 benchmarks), CSAT is <85% (the canonical "our 1-star-reviews mention slow-response-time" anti-pattern per Gorgias CSAT 2024 + Help Scout CSAT 2024 benchmarks), ticket-deflection-rate is <10% (the canonical "we're answering every 'where is my order' question manually" anti-pattern per Gorgias Automate 2024 + Klaviyo Customer Agent 2024 benchmarks), the operator cannot answer "what is our CSAT-by-channel-by-cohort today?" within 30 seconds (the canonical "we don't have a CX-dashboard" anti-pattern), chargeback-deflection-rate is <30% (the canonical "the customer called their bank because they couldn't reach us" anti-pattern per Chargeflow 2024 + Move #23 fraud benchmarks), or upsell-cross-sell-rate is <5% of tickets (the canonical "we're missing $50k+/year in upsell-recovery from CS-tickets" anti-pattern per Gorgias 2024 + Triple Whale 2024 benchmarks). Move #24 is the **canonical CX-ops foundation** after Move #11 (AI customer service) + Move #20.2 (NPS) + Move #21 (3PL) + Move #22 (checkout) + Move #23 (fraud) — CX-ops is the layer the operator pulls once the rest of the stack is shipping volume; without the prior moves shipping volume, the CX-platform has insufficient tickets to amortize the platform fee.

You have:

- **Shopify (or Ikas / BigCommerce / WooCommerce / headless) DTC store** with admin API access AND the Orders API + Customers API + Products API + Inventory API + Refunds API enabled (Shopify Plus OR Shopify Advanced with the necessary scopes for the helpdesk-platform-app to read order-context in the ticket-view).
- **≥500 orders/month processed in the last 30 days** AND **≥$50 AOV** AND **≥$120k ARR** AND **≥100 tickets/month** — below these thresholds, defer and use Shopify Inbox (free) + a shared Gmail-inbox + Google-Sheets-Ticket-Log until volume justifies the platform fee. The canonical pre-shipping signal is "the cost of the platform fee exceeds the cost of the slow-response-time + low-FCR + low-CSAT-driven-churn" — at <100 tickets/mo Path A (Shopify Inbox free + a shared Gmail inbox) is sufficient.
- **Move #1 (cart recovery) + Move #5 (Klaviyo+Postscript) + Move #6 (Triple Whale attribution) shipped and producing real data** — the helpdesk-platform reads Klaviyo-customer-context (open-email-rate + click-rate + last-open + last-click + purchase-history) + Triple-Whale-cohort-LTV (per-customer-source-per-customer-LTV-band) + Shopify-order-context (order-status + tracking + last-refund + last-exchange) to render a 360-degree-customer-view inside the ticket-view. Without these upstream, the helpdesk-agent is answering tickets blind to the customer-journey.
- **Move #11 (AI customer service automation) shipped OR being-shipped-this-tick** — the helpdesk-platform is the substrate for Gorgias-Automate + Klaviyo-Customer-Agent + Gorgias-Macros. Without the platform, the AI-deflection-engine has nowhere to fire.
- **A documented CX-policy** — first-response-time-SLA + resolution-time-SLA + ticket-routing-rules + macro-library-coverage + escalation-rules + after-hours-rotation + escalation-to-engineering + escalation-to-shipping-3PL. Without this, the helpdesk-platform has no governance to optimize against. The canonical template is: `first_response_sla = <1 hr for tickets tagged "cs-detractor-urgent" OR "cs-fraud-chargeback-urgent", <4 hr for tickets tagged "cs-shipping-status", <8 hr for tickets tagged "cs-product-question"; resolution_sla = <24 hr for "cs-detractor-urgent" OR "cs-fraud-chargeback-urgent", <48 hr for "cs-shipping-status" OR "cs-product-question", <72 hr for "cs-returns-request"; routing = Shopify-order-context customer-tier → senior-CS-agent, paid-cohort customer → mid-CS-agent, organic-cohort customer → junior-CS-agent; macro_coverage_target = 80%+ of "cs-shipping-status" + "cs-product-question" + "cs-tracking-number" handled by macros + automation; escalation = >24 hr unresolved → senior-CS-agent, >48 hr unresolved → operations-manager, >72 hr unresolved → CX-team-lead`.

## What "best in class" looks like

A best-in-class customer-service-operations-platform has all 5 of these properties:

1. **5-pillar helpdesk-platform-selection framework.** Path A (Shopify Inbox free + shared Gmail-inbox + Google-Sheets-Ticket-Log + Slack-channel) for <$1M GMV <100 tickets/mo; Path B (Gorgias Pro $300/mo DEFAULT for $1M–$5M GMV Shopify-DTC apparel/beauty/home with Shopify-order-context-in-ticket-view + Gorgias-Automate + Klaviyo-context-injection + Gorgias-Macros-rule-library + Slack-360-bot); Path B+ (Gorgias Advanced $750/mo OR Help Scout Plus $65/user/mo for $3M+ GMV non-Shopify brands needing Beacon-help-center + Docs-knowledge-base + AI-Answer); Path C (Zendesk Suite Pro $115/agent/mo OR Kustomer $89/agent/mo for $5M+ GMV brands needing multi-brand-support + Sunshine-CRM + Skill-based-routing + multi-channel-inbox); Path D (Front $65/seat/mo + Gorgias-as-engine for the front-end + Help-Scout-AI-Answer for the back-end for $5M+ GMV brands needing collaborative-email-inbox-with-shared-drafts-and-internal-comments). Engine selection drives everything downstream — wrong path picks cost 5–10× over a 12-month horizon.

2. **5-pillar framework applied in this order:**

   - **Pillar 1 — Helpdesk-platform-selection + Shopify-integration + Klaviyo-context-injection (Week 1, 6 hr).** Path A Shopify-Inbox + Gmail-inbox + Google-Sheets-Ticket-Log for <$1M GMV; Path B Gorgias Pro $300/mo DEFAULT for Shopify-DTC; Path B+ Gorgias Advanced $750/mo OR Help Scout Plus $65/user/mo for non-Shopify; Path C Zendesk Suite Pro $115/agent/mo OR Kustomer $89/agent/mo for enterprise. Wire Shopify-orders-API + Shopify-customers-API + Shopify-products-API + Shopify-refunds-API to the helpdesk-platform so the agent sees order-status + tracking-number + last-refund + last-exchange + customer-LTV + Klaviyo-engagement-score inside the ticket-view. Without the order-context-injection, the agent is answering tickets blind to the customer-journey. Verify: a test-ticket-from-a-real-customer renders the full 360-degree-customer-view in <2 seconds.

   - **Pillar 2 — Ticket-routing + triage + macro-library + SLA-management (Week 1–2, 8 hr).** Define ticket-tags (`cs-shipping-status` + `cs-tracking-number` + `cs-product-question` + `cs-returns-request` + `cs-exchange-request` + `cs-refund-request` + `cs-damaged-product` + `cs-wrong-product` + `cs-missing-product` + `cs-payment-issue` + `cs-account-issue` + `cs-subscription-pause` + `cs-subscription-cancel` + `cs-loyalty-question` + `cs-discount-question` + `cs-detractor-urgent` + `cs-fraud-chargeback-urgent`). Build a macro-library (≥50 macros) covering the top 20 ticket-types: macro `shipping-status-template` (Klaviyo + Gorgias + Triple Whale context + tracking-link + estimated-delivery-date) + macro `product-question-template` (Shopify-products-API + knowledge-base-link) + macro `refund-request-template` (Shopify-refunds-API + refund-amount + refund-method + processing-time) + macro `exchange-request-template` (Shopify-exchanges-API + return-shipping-label + new-product-shipping-label) + macro `subscription-pause-template` (Recharge-or-Skio-API + pause-duration-options + resume-date-calculation) + macro `subscription-cancel-template` (Recharge-or-Skio-API + cancel-effective-date + cancel-reason-options + cancel-feedback-survey). SLA-management: first-response-time <1 hr for "cs-detractor-urgent" + "cs-fraud-chargeback-urgent", <4 hr for "cs-shipping-status" + "cs-tracking-number" + "cs-payment-issue", <8 hr for "cs-product-question" + "cs-account-issue" + "cs-loyalty-question"; resolution-time <24 hr for urgent, <48 hr for normal, <72 hr for low-priority. Routing-rules: Shopify-order-context-customer-tier → senior-CS-agent, paid-cohort-customer → mid-CS-agent, organic-cohort-customer → junior-CS-agent; ticket-priority "cs-detractor-urgent" → senior-CS-agent queue + Slack-alert-to-CX-lead; "cs-fraud-chargeback-urgent" → senior-CS-agent + Slack-alert-to-CX-lead + Gorgias-deflection-message-auto-sent (per Move #23 fraud playbook).

   - **Pillar 3 — AI-powered-ticket-deflection + automation-rules + Gorgias-Automate + Klaviyo-Customer-Agent (Week 2–3, 10 hr).** Build the AI-deflection-rule-library: ≥20 automation-rules covering the top deflection-paths. Rule `auto-reply-shipping-status`: when ticket-body-contains "where is my order" + Shopify-order-status = "in_transit" OR "delivered" + no-prior-ticket-on-this-order → auto-reply-with-tracking-link + estimated-delivery-date + Gorgias-macro-shipping-status-template. Rule `auto-reply-product-question`: when ticket-body-contains-product-name + Help-Scout-Docs-has-matching-article → auto-reply-with-knowledge-base-link + Gorgias-macro-product-question-template. Rule `auto-reply-subscription-pause`: when ticket-body-contains "pause my subscription" + customer-has-active-Recharge-subscription → auto-reply-with-Recharge-customer-portal-link + pause-duration-options. Rule `auto-reply-subscription-cancel`: when ticket-body-contains "cancel my subscription" + customer-has-active-Recharge-subscription → auto-reply-with-Recharge-customer-portal-link + cancel-feedback-survey + save-cancel-reason-to-Klaviyo-segment. Rule `auto-deflect-repeat-question`: when same-customer-asks-same-question-twice-in-7d → auto-reply-with-knowledge-base-link + tag "cs-repeat-question". Verify: the AI-deflection-rule-library deflects ≥30% of tickets in the first 30 days post-launch.

   - **Pillar 4 — Multi-channel-integration + Shopify + Klaviyo + Recharge + Loop-returns + Triple-Whale + Slack + ShipBob-WMS (Week 3–4, 8 hr).** Wire the helpdesk-platform to all upstream data sources so the agent sees the full 360-degree-customer-view: Shopify (orders + customers + products + refunds + inventory) + Klaviyo (engagement-score + open-email-rate + last-open + last-click + flow-history + segment-membership) + Recharge OR Skio (subscription-status + subscription-cancel-reason + next-billing-date + pause-history + pause-duration) + Loop-returns OR ReturnGO (return-status + return-reason + return-tracking + return-to-stock-event) + Triple Whale (cohort-LTV-band + acquisition-source + cohort-LTV-vs-baseline) + Smile.io OR Yotpo (loyalty-points-balance + loyalty-tier + referral-code + referral-history) + Slack (Slack-360-bot-for-internal-assign + escalation-alert + customer-journey-snapshot) + ShipBob OR ShipMonk OR Stord (3PL-WMS-status + last-shipment + last-delivery-confirmation + return-to-warehouse-event). ## Common pitfalls (15 from real builds)

1. **No macro-library built.** Answering every ticket-from-scratch is the canonical "we're slow-and-inconsistent" anti-pattern — first-response-time-stays-at-4–8 hr + FCR-stays-at-40–55% + CSAT-stays-at-70–80%. **Fix:** build the macro-library ≥50 macros covering the top-20-ticket-types in Week 1. Each macro injects Klaviyo-customer-context + Shopify-order-context + Triple-Whale-cohort-LTV. Verify: macro-coverage ≥80% of the top-20-ticket-types. *Cost of ignoring: $30k–$100k Year-1 lost-revenue-or-operator-time per Gorgias 2024 + Help Scout 2024 benchmarks.*

2. **No AI-deflection-rule-library built.** Letting every ticket reach-an-agent is the canonical "we're drowning-in-tickets" anti-pattern — ticket-deflection-rate-stays-at-0–10% + operator-time-saved-stays-at-0–2-hr/wk + first-response-time-stays-at-4–8 hr. **Fix:** build the AI-deflection-rule-library ≥20 rules in Week 2. Each rule auto-replies with a macro-injected or auto-detects-detractor + auto-routes to senior-CS-agent. Verify: ticket-deflection-rate ≥30% in the first 30 days. *Cost of ignoring: $20k–$80k Year-1 lost-operator-time per Gorgias Automate 2024 + Klaviyo Customer Agent 2024 benchmarks.*

3. **No multi-channel-integration wired.** Answering-tickets-blind-to-the-customer-journey is the canonical "we-don't-know-if-this-customer-is-a-repeat-buyer-or-a-detractor" anti-pattern — CSAT-stays-at-70–80% + missed-chargeback-deflection + missed-upsell-cross-sell. **Fix:** wire all upstream data sources (Shopify + Klaviyo + Recharge + Loop-returns + Triple-Whale + Smile + Slack + ShipBob-WMS) into the ticket-view-in-<2-sec. Verify: a test-ticket-renders-the-360-degree-customer-view-in-<2-sec. *Cost of ignoring: $20k–$80k Year-1 lost-CSAT-driven-retention + missed-chargeback-deflection per Gorgias 2024 + Triple Whale 2024 benchmarks.*

4. **No SLA-management configured.** Without-first-response-time-SLA + resolution-time-SLA the agent has-no-accountability-for-speed — first-response-time-stays-at-8–24 hr + CSAT-drops-to-70–80%. **Fix:** configure first-response-time-SLA <1 hr urgent / <4 hr normal / <8 hr low + resolution-time-SLA <24 hr urgent / <48 hr normal / <72 hr low.
