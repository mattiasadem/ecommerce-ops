## When to use this skill

Use this skill when the operator is doing **≥$1M GMV** AND the financial-operations curve has started to compound: the operator has **≥$10k/mo ad-spend** but cannot answer "what is our per-channel contribution-margin AND payback-period AND cash-conversion-cycle?" within 60 seconds (the canonical "we know we spent $20k on Meta last month but we don't know if it returned more than $20k in contribution-margin after COGS + 3PL + returns + sales-tax" anti-pattern per Triple Whale 2024 + Polar 2024 + Kruze Consulting DTC Benchmarks 2024 benchmarks), the operator has **3+ sales-channels** (Shopify + Amazon + Faire + TikTok-Shop + wholesale + retail) but cannot see per-channel-P&L-bridge (the canonical "Amazon shows up as profitable but is cannibalizing 35% of Shopify-DTC-margin" anti-pattern per ecommerce-channel-p-and-l-2024 + marketplace-expansion benchmarks), the operator has **multi-currency revenue** (EU + UK + Canada + AU + LATAM) but reconciles FX manually at month-end (the canonical "our multi-currency revenue looks great in USD until you account for FX + payment-fees + VAT-withholding + marketplace-facilitator-fees" anti-pattern per stripe-multi-currency-2024 + netsuite-fx-2024 + sage-intacct-fx-2024 + shopify-markets-2024 benchmarks), the operator has **>1.5x revenue multiple in accounts-receivable outstanding** from B2B wholesale orders (the canonical "Faire + Ankorstore NET-60-terms mean we wait 60-90 days to get paid and we ran out of cash" anti-pattern per Faire 2024 + Ankorstore 2024 + b2b-wholesale benchmarks + ecommerce-ar-financing-2024 + ecommerce-cash-conversion-2024), the operator has **25-day month-end-close cycle** with 3+ person finance team (the canonical "we close the books in the last week of the next month" anti-pattern per QuickBooks Online Advanced 2024 + Sage Intacct 2024 + Netsuite 2024 + dtc-cfo-benchmarks-2024 benchmarks showing operators who move to QuickBooks Online Advanced + Ramp + Cin7 close in ≤5 days vs the 25-day manual baseline), the operator has **multi-entity structure** (US-LLC + UK-Ltd + EU-GmbH + DE-GmbH + CA-Corp + AU-Pty-Ltd) without automated intercompany-elimination (the canonical "we forgot to eliminate the intercompany-transfer-pricing in the consolidation and our GAAP-entity-consolidated-P&L is off by 28%" anti-pattern per Netsuite Multi-Entity 2024 + Sage Intacct Multi-Entity 2024 + Anaplan Multi-Entity 2024 + Drivetrain 2024 benchmarks), the operator has **inventory-financing or working-capital line-of-credit** but cannot project cash-needs 13-weeks-out (the canonical "we draw ecommerce-credit-line-2024 benchmarks), the operator is **raising capital** but the board-deck-cap-table-inventory-FPA-overlay is built in Excel every quarter (the canonical "we spent 40 hours cube-board-deck-2024 + runway-board-deck-2024 benchmarks), the operator has **>100 employees or contractor-payouts** but expense-reports take 21+ days to close (the canonical "contractor-payouts are 30-days-late and they keep asking why" anti-pattern per Ramp 2024 + Brex 2024 + airbase-2024 + spendesk-2024 + harold-cfo-2024 benchmarks), or the operator has **>4-person finance team** but is still doing reconciliation in Excel (the canonical "we have a controller who closes in Excel and a CFO who trusts nothing from the controller and rebuilds everything in Google Sheets" anti-pattern per cube-cfo-2024 + pigment-cfo-2024 + anaplan-cfo-2024 + jirav-cfo-2024 + drivetrain-cfo-2024 + runway-cfo-2024 benchmarks). Move #40 is the **canonical next step** in the financial-operations track after Move #21 (sales-tax-VAT) + Move #22 (international-expansion) are live OR the operator is doing ≥$1M GMV AND has multi-channel-revenue OR multi-currency-revenue OR multi-entity-structure OR inventory-financing OR ≥3-person finance team OR is preparing for a fundraise-and-needs-board-deck-cap-table-inventory-FPA-overlay.

You have:

- **Shopify (or Ikas / BigCommerce / WooCommerce / headless) DTC store** with admin API access + Orders API + Customers API + Inventory API + Refunds API + Transactions API enabled.
- **Stripe (or Braintree / Adyen / Authorize.Net / PayPal Payments Pro) payment-gateway** with API access + Connect (if selling wholesale) + Revenue-Recognition API (if ASC-606-compliance-needed).

Do **not** use it when:

- **<$1M annual GMV AND <$10k/mo ad-spend AND <3 sales-channels AND no multi-currency-revenue AND no multi-entity-structure AND <3-person finance team.** Below these thresholds, defer Move #40 and use Shopify-native-financials + QuickBooks Online Simple Start + Ramp Starter + Mercury for cash-management + manual-Excel-P&L-tied-to-shopify-export. ## What "best in class" looks like

A best-in-class DTC-financial-operations engine for a $1M-$10M GMV multi-channel-multi-currency DTC brand has all 5 of these properties:

1. **5-pillar financial-operations-engine framework.** Path A (Shopify-native-financials + QuickBooks Online Simple Start $30/mo + Ramp Starter $0/user + Mercury $0 + Manual-Excel-P&L + fractional-bookkeeper $500-$1,500/mo) for <$1M GMV solo-operator; Path B (QuickBooks Online Advanced $200/mo + Sage Intacct $400-$1,000/mo + Ramp $0-$15/user/mo + Mercury $0 + Cin7 $325-$1,000+/mo + Stripe + Triple Whale + Klaviyo + Gorgias Integrations + Manual-board-deck-with-Google-Slides) DEFAULT for $1M-$5M GMV multi-channel-3PL; Path B+ (Sage Intacct $400-$1,000/mo + Cin7 Omni $1,000+/mo + Ramp $0-$15/user/mo + Brex $0-$12/user/mo + Mercury Treasury + Automated-board-deck-via-Mosaic-or-Jirav) for $3M-$10M GMV multi-currency-multi-warehouse; Path C (NetSuite $1,000-$5,000+/mo + Anaplan $1,500-$5,000+/user/mo + Cube $750-$3,000/user/mo + Pigment $1,000-$4,000/user/mo + Drivetrain $1,500-$5,000+/user/mo + Cin7 Omni $1,000+/mo + Carta + Pulley + Mercury Treasury + Per-Entity-Multi-Currency-Consolidation + ASC-606-Automated-Revenue-Recognition) for $10M+ GMV multi-entity-multi-currency-multi-warehouse brands with $5k+/mo finance-team OR pre-IPO-readiness OR PE-portfolio-consolidation-readiness; Path D (Build-it-yourself with NetSuite + Airbase + Tipalti + Modern Treasury + Carta + Anaplan + custom-integration-build) for $50M+ GMV brands that need bespoke consolidation-and-FP&A. Engine selection drives everything downstream — wrong path picks cost 5-10× over a 12-month horizon and produce operator-head-only-FP&A which doesn't scale.

2. **5-pillar framework applied in this order:**

   - **Pillar 1 — Chart-of-accounts + contribution-margin-dashboard + per-SKU per-cohort P&L-bridge (Week 1-2, 6 hr).** Document the canonical 6-tier chart-of-accounts: Tier 1 revenue (per-channel per-cohort per-SKU per-market per-currency) / Tier 2 COGS (per-SKU landed-cost = product-cost + inbound-freight + duty + 3PL-fulfillment-cost per-order + packaging-cost per-order + product-insert-cost per-order + gift-wrap-cost per-order + per-payment-processing-fees + per-sales-tax-paid-on-purchase) / Tier 3 contribution-margin-1 (revenue − COGS − payment-fees − sales-tax-paid) / Tier 4 contribution-margin-2 (contribution-margin-1 − 3PL-cost − shipping-label-cost − returns-processing-cost − sales-tax-collected-remitted-cost − marketplace-fee) / Tier 5 contribution-margin-3 (contribution-margin-2 − ad-spend − email/SMS-cost − creative-cost − influencer-cost − agency-fee) / Tier 6 contribution-margin-4 (contribution-margin-3 − operating-cost = rent + salaries + software + insurance + legal + accounting + CX-cost). Wire Shopify + Stripe + Klaviyo + Gorgias + Triple Whale + Cin7 to feed the chart-of-accounts via API-webhooks (Shopify Admin API → QuickBooks Online Advanced + Cin7; Stripe → QuickBooks Online Advanced + Revenue-Recognition; Klaviyo + Postscript → revenue-attribution-cogs-line; Gorgias → CX-cost-line; Triple Whale + Polar + Northbeam → ad-spend-line + cohort-revenue-line). Build the **per-channel per-cohort per-SKU contribution-margin-dashboard** that updates daily (looker-studio-or-metabase-or-mode-or-anaplan-or-jirav-or-cube-or-runway). Without this Pillar 1, the operator cannot measure the lift from Move #6 + Move #12 + Move #21 + Move #22 + Move #26 + Move #27 etc.

   - **Pillar 2 — Unit-economics-calculator + COGS-allocation + break-even-model + LTV-CAC-payback-engine (Week 2-3, 8 hr).** Build the canonical 5-axis unit-economics-model: Axis 1 per-order-contribution-margin (CM-1 + CM-2 + CM-3 + CM-4 per order per channel per cohort) / Axis 2 per-cohort-LTV (12-month-cohort-LTV-by-acquisition-channel + 24-month-cohort-LTV-by-acquisition-channel + per-tier-LTV + per-product-LTV + per-market-LTV) / Axis 3 CAC + payback (blended-CAC + per-channel-CAC + per-cohort-CAC + per-product-CAC + per-market-CAC + blended-payback + per-channel-payback + per-cohort-payback) / Axis 4 working-capital (inventory-days-on-hand + AR-days-outstanding + AP-days-outstanding + cash-conversion-cycle) / Axis 5 magic-number + rule-of-40 (Magic-Number = (Q4-revenue − Q3-revenue) × 4 ÷ Q4-sales-and-marketing-spend; Rule-of-40 = revenue-growth-rate + EBITDA-margin). Wire Stripe + Klaviyo + Gorgias + Triple Whale + Cin7 + NetSuite-or-QuickBooks-Online-Advanced + ShipBob to feed the calculator via API-webhooks. The LTV-CAC-payback-engine surfaces the 12-25% per-channel-LTV-dispersion per Triple Whale 2024 + Northbeam 2024 + Polar 2024 benchmarks that drives every marketing-spend-decision. Without this Pillar 2, the operator makes marketing-spend-decisions on gross-revenue + gross-CAC without the per-cohort-per-channel-per-SKU net-economics.

   - **Pillar 3 — Cash-flow-forecast + 13-week-rolling-forecast + working-capital-engine + multi-currency-FX-revaluation (Week 3-5, 10 hr).** Build the canonical 13-week-cash-forecast that updates weekly (using Shopify + Stripe + Ramp + Mercury + Cin7 + NetSuite + Triple Whale + Klaviyo + Gorgias webhook-data) — the forecast shows week-by-week cash-position that lets the operator know 8-12 weeks in advance if cash is going to run out. The forecast includes: (a) **inflows** = Shopify + Faire + Ankorstore + Amazon-Settlement + TikTok-Shop-Settlement + Stripe-Billing-renewal + B2B-AR-inflow + other-channel-inflow, per-week per-source / (b) **outflows** = inventory-replenishment-PO-week + payroll + ad-spend-week + 3PL-bill-pay-week + Klaviyo + Postscript + Gorgias + Gorgias-CX-costs + Triple-Whale + Ramp-Bill-Pay-vendor-payments + sales-tax-VAT-remittance-quarterly + monthly-rent + monthly-software + monthly-insurance + monthly-legal + monthly-accounting + monthly-contractor-payouts / (c) **FX-revaluation** = per-currency-week cash-balance + per-currency-week forward-FX-rate × per-currency-week cash-balance / (d) **liquidity-events** = expected-AR-payments + expected-credit-line-drawdowns + expected-inventory-financing + expected-equity-injection + expected-PPP-loan-or-SBA-loan-or-fundraise. Wire Sage Intacct / NetSuite / Cube / Runway / Jirav / Anaplan / Drivetrain / Pigment / Mosaic to run the forecast. Without this Pillar 3, the operator runs out of cash in Q1 even with positive-P&L per ecommerce-13-week-cash-forecast-2024 + ecommerce-cash-conversion-2024 + ecommerce-runway-2024 benchmarks showing 50-70% of DTC-brand-failures are cash-flow not P&L failures.

   - **Pillar 4 — FP&A-automation + month-end-close-automation + AP-AR-inventory-reconciliation + ASC-606-revenue-recognition (Week 5-8, 12 hr).** Automate the canonical 5-step month-end-close: Step 1 bank-feed-reconciliation (Ramp + Brex + Mercury + Mercury-Treasury + Stripe-Balance + Shopify-Payments-Balance + Wise + PayPal-Balance + Amazon-Pay-Balance + Faire-Payable-Balance → QuickBooks Online Advanced + Sage Intacct + NetSuite via Plaid-and-merchant-feed-API) / Step 2 AP-reconciliation (Ramp + Brex + Bill.com + Airbase + Tipalti + Spendesk + Payhawk → QuickBooks Online Advanced + Sage Intacct + NetSuite via vendor-bill-API) / Step 3 AR-reconciliation (Faire + Ankorstore + Amazon + TikTok-Shop + Stripe-Billing → QuickBooks Online Advanced + Sage Intacct + NetSuite via receivable-API) / Step 4 inventory-reconciliation (Shopify + Amazon + 3PL + Cin7 + LitCommerce + TradeGecko + Fishbowl → QuickBooks Online Advanced + Sage Intacct + NetSuite + inventory-valuation-engine via inventory-API) / Step 5 sales-tax-VAT-reconciliation (Avalara + TaxJar + Stripe Tax + Shopify Markets Pro + Vertex + Sovos → QuickBooks Online Advanced + Sage Intacct + NetSuite + sales-tax-payable-account). Plus ASC-606-revenue-recognition-engine (Stripe Revenue Recognition + NetSuite Revenue Recognition + Sage Intacct Revenue Recognition → recognize-revenue-when-performance-obligation-satisfied; for subscriptions, recognize ratably over the subscription-period; for one-time-purchases, recognize at ship-time; for marketplace-facilitator-tax-collected, recognize as liability not revenue per ASC-606-2024 + ecommerce-revenue-recognition-2024 + ecommerce-cogs-recognition-2024 benchmarks). Without this Pillar 4, the operator runs a 25-day month-end-close with 3-5 person finance-team per QuickBooks Online Advanced 2024 + Sage Intacct 2024 + Netsuite 2024 + dtc-cfo-handbook-2024 + bf80-CFO-Handbook-2024 benchmarks vs ≤5 days with the canonical 5-step automation.

   - **Pillar 5 — Board-deck + investor-update + cohort-P&L-overlay + Triple-Whale-FP&A-cohort-bridge + cap-table-management (Week 8-12, 12 hr).** Build the canonical quarterly-board-deck using a financial-orchestration-platform (Cube / Runway / Drivetrain / Jirav / Mosaic / Pigment / Anaplan / LiveFlow / Fintastic) that pulls real-time-data-from-Shopify + Stripe + Triple-Whale + NetSuite-or-QuickBooks-Online-Advanced + Cin7 + Ramp + Carta + Cap-table-data + Revenue-Recognition-data + Sales-Tax-Liability-data. The board-deck includes: (a) **executive-summary** = revenue + gross-margin + contribution-margin + EBITDA + runway + burn-multiple + Rule-of-40 + Magic-Number + cash-conversion-cycle + LTV-CAC + per-channel-CAC + per-cohort-CAC, updated automatically / (b) **P&L-detail** = per-channel per-cohort per-SKU per-month-per-quarter revenue + COGS + contribution-margin-bridge / (c) **cohort-overlay** = 12-month-cohort-LTV-by-acquisition-channel + per-cohort-CAC + per-cohort-payback + per-cohort-cash-conversion / (d) **cash-position** = 13-week-cash-forecast + per-currency-balance + per-LOC-drawdown-status + per-credit-line-utilization + per-AR-aging + per-AP-aging / (e) **cap-table** = per-share-class + per-round-dilution + fully-diluted-share-count + per-option-pool + per-employee-equity-grant-amortization-on-P&L per ASC-718 / IFRS-2 / Carta 2024 + Pulley 2024 + Ledgy 2024 + Shareworks 2024 benchmarks / (f) **next-quarter-forecast + annual-plan** = driver-based-forecast-by-revenue-line + COGS-line + opex-line + capex-line + per-quarter-target + per-month-target. The canonical 12-pillar-board-deck updates automatically every Monday morning so the founder + CFO + controller + board-members can see real-time-finance without anyone touching Excel. Visible.vc + Quaestor + Eqvista + Carta auto-generate the investor-update-template.
